1. JavaScript bug breaks the interest form

Critical · Line 682

The workshop_id value on line 682 is a bare UUID without quotes: workshop_id: 019d57ee-a544-7223-afcb-c6422d3f7fb3. JavaScript interprets the hyphens as subtraction operators, producing NaN. The POST body sends garbage, so no interest submission actually works.

High ImpactCode Fix
Fix Wrap the UUID in quotes: workshop_id: '019d57ee-a544-7223-afcb-c6422d3f7fb3'. Test the form after deploying — every submission since launch has been broken.

2. No social proof anywhere on the page

High Impact · Trust

The page asks MSPs to spend EUR 9,999–15,500 but offers zero evidence that the team has delivered results. No client logos, no testimonials, no case study numbers, no "X MSPs served" counter. The Showcases section lists what Dxfferent built, but there is no third-party validation. At this price point, buyers need proof.

High ImpactUX
Fix Add a section between Showcases and the mid-page CTA with 2–3 short testimonials from MSP owners. Even one quote with a name, role, and company adds credibility. If testimonials are not available yet (Cohort 1), add logos of MSPs the team has worked with, or a stat like "Built AI systems for 12 MSPs in 2025."

3. Hero headline does not name the buyer's outcome

High Impact · Copy

"Bouw een compliant AI systeem voor jouw MSP. In 3 dagen." tells people what they will do, not what they will get. The subtext above the headline carries the real value prop (more value without growing headcount), but it is styled as muted pre-text that most visitors will skip.

High ImpactUX
Fix Flip the hierarchy. Lead with the outcome in the H1: something like "Lever 3x meer waarde met hetzelfde team. Zonder eigen developers." Push the "3 dagen / compliant AI systeem" detail to the subtitle. The pre-text sentence above the H1 can stay, but the H1 itself should sell the result, not the activity.

4. No urgency or scarcity mechanism

Medium Impact · Conversion

The booster section says "Nog 6 plekken beschikbaar" as static text. There is no countdown timer, no dynamic seat counter, and no early-bird deadline. The date (May 18) is about 5 weeks away. Static scarcity text that never changes trains buyers to wait.

Medium ImpactUX
Fix Add a countdown timer in the hero or above the pricing section showing days until the workshop. If possible, pull the real seat count from the registration API. If not, at least add an early-bird deadline: "Schrijf in voor 1 mei en ontvang X" to create a reason to act now.

5. Pricing cards lack feature differentiation

Medium Impact · UX

The three tiers (Workshop / Begeleiding / Transformatie) describe what is included in paragraph form. There is no checklist, no visual comparison, and no clear "this tier adds X over the previous one" structure. At a glance, a visitor cannot tell why Transformatie costs EUR 5,500 more than Workshop.

Medium ImpactUX
Fix Replace the paragraph descriptions with a bulleted feature list per card, or add a comparison table below the cards. Use checkmarks and dashes so the delta between tiers is visible at a glance. Bold the features that are unique to higher tiers.

6. Missing meta tags and OG data

Medium Impact · SEO / Sharing

The <head> contains only a <title> and font imports. There is no meta description, no Open Graph tags, no Twitter card tags, and no favicon. When someone shares this link on LinkedIn or Slack, it will show a blank preview — a missed opportunity for a page designed to attract MSP decision-makers.

Medium ImpactCode Fix
Fix Add meta description, og:title, og:description, og:image, and og:url tags. Create a 1200x630 social share image with the workshop title and date. Add a favicon. This takes 10 minutes and makes every share link look professional.

7. Mobile breakpoint ordering is wrong

Medium Impact · Responsive

The 768px breakpoint stacks all grids to single column, which is correct. But the 1024px breakpoint (line 182) is declared after the 768px one. Since 768px rules still match at 769–1024px, the 1024px rules override them, putting outcomes and showcases back into 2 columns on tablets. The cascade is backwards — mobile-first media queries should go from small to large.

Medium ImpactCode Fix
Fix Swap the media query order: @media(max-width:1024px) first, then @media(max-width:768px). Or switch to min-width mobile-first breakpoints. Test on iPad (1024px) and phone (375px) to confirm grids stack correctly.

8. No sticky navigation or back-to-top

Low Impact · UX

The page has 14 sections and runs long. There is no navigation bar, no anchor menu, and no back-to-top button. A visitor who scrolls past the pricing section has no way to jump back to the hero CTA without manual scrolling. The "Meer informatie ontvangen" link in the hero points to #interest-form, but there is no way to get back up.

Low ImpactUX
Fix Add a minimal sticky bar at the top with the Dxfferent logo, a one-line value prop, and a "Reserveer" button. This keeps the CTA visible at all scroll depths. A floating back-to-top button (bottom right, appears after scrolling 600px) is a low-effort alternative.

9. Interest form has no feedback on input validation

Low Impact · UX

The email field uses native required and type="email" validation, but the only error handling in the JS is a generic red box. If the API returns a validation error for the email field, it replaces the error div's textContent — but the original error HTML with the mailto fallback link is lost permanently. Also, there is no loading spinner, just the text "Versturen..." on a disabled button.

Low ImpactCode Fix
Fix Use innerHTML instead of textContent on line 699 so the email fallback link is preserved. Or better: keep the generic error div as-is, and add a separate inline error span below the email input for field-specific errors. Add a simple CSS spinner or pulse animation to the button during submission.

10. Accessibility gaps

Low Impact · Code Quality

Several issues: the hero section has no <header> landmark, <section> elements lack aria-label or headings for screen readers, the pain-quote class uses a styled div instead of a <blockquote>, zone color indicators rely solely on color (no text label for colorblind users), and the FAQ details/summary pattern hides the marker but does not add an aria-expanded attribute.

Low ImpactCode Fix
Fix Wrap the hero in <header role="banner">. Add aria-label to each <section>. Use <blockquote> for pain quotes. Add a text label ("Vrij" / "Pseudonimiseer" / "Lokaal") next to each zone color indicator (already done in <h3>, so the indicator dot is decorative and should get aria-hidden="true"). The FAQ is fine functionally since <details> handles its own aria state natively.
