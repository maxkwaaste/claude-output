De vijf veranderingen

Uit brainstorm 30 april 2026. Elke verandering met de bijbehorende beslissing erbij (zoals afgestemd in deze sessie).

1. Title-veld weg, hook templates erin
EDITOR · OPENER
Het huidige title-veld voelt geforceerd. Vervangen door één "Opener" veld met een picker-knop ernaast die de 40 bestaande hook templates uit seed.json toont, gefilterd op het geselecteerde content-type. Klik op template = veld vult zich met starter-tekst die de consultant aanpast. Interne identifier wordt afgeleid uit de eerste 60 chars van de opener.
Beslissing: bestaande hook-templates uit seed.json
2. MSP Journey context editable + apart tab
EDITOR + NIEUWE TAB
Topics in cellen mogen door alle 5 consultants worden toegevoegd/verwijderd. Twee toegangspunten: dedicated tab voor masterview en bulk-onderhoud, plus inline edit in de artikel-schrijver wanneer je een cel hebt aangeklikt. Open editing voor iedereen, met audit log voor accountability en undo-knop voor "oeps".
Beslissing: open voor iedereen + audit log + undo
3. Vrije tekst krijgt metadata-velden
EDITOR · SIDE PANEL
Bij vrije tekst (niet uit de matrix gestart) komen optionele velden: Onderwerp, Domein, MSP-niveau (multi-select, bv. "2 & 3"), Type. Allemaal optioneel. Als consultant ze leeg laat, vult AI ze later in (Phase 2.5). Velden in een collapsible side panel rechts in de editor, niet boven het schrijfveld.
AI-fill: expliciete knop + auto-suggest op save
4. Matrix als primaire content-library interface
DE KILLER · TAB 1
De huidige topic-list-with-filters wordt vervangen door een 8×5 matrix die de Dxfferent MSP Journey reflecteert. Cellen tonen twee dimensies tegelijk: kleur = Joshua's prioriteit (1-5 schaal), badge rechtsboven = aantal artikelen al geschreven. Klik op cel → modaal met content-type keuze → editor met domain+level+type prefilled, hook picker direct open. Voor MVP: priority-data is fake (2-3 = hoog), Joshua kan later admin-modus krijgen.
Klik-flow: tab-bar in editor (mijn voorkeur)
5. Bibliotheek-overzicht (source posts + derivatives + status)
NIEUWE TAB · OPERATIONEEL
Flat lijst van alle source posts (consultant ruwe input) met daaronder de afgeleide content (LinkedIn-versie, blog-versie, etc.), status per derivative (draft/scheduled/posted), live URL als gepost en performance-metrics (likes/views/comments + datum laatste update). Drie numerieke velden, handmatige invoer in MVP, LinkedIn API komt later.
Performance: handmatig MVP, 3 numerieke velden

Domein-mismatch: kritiek punt

De publieke MSP Journey op dxfferent.com gebruikt 8 domeinen × 5 maturity-levels = 40 cellen. De huidige seed.json heeft maar 5 domeinen, geen level-veld, en verkeerde namen. Voor de matrix-UI moeten alle 64 bestaande topics opnieuw worden ingedeeld in dit grid.

Migratie-aanpak (aanbevolen) B: AI doet eerste pass, Joshua reviewt. Claude leest elk topic, mapt naar één van 40 cellen op basis van titel + context, levert CSV op. Joshua keurt goed in 30 min. Schaalt straks ook voor nieuwe topics. Alternatief A (Joshua handmatig) is fine maar duurt langer en moet hij zelf doen. Alternatief C (lege levels) blokkeert de matrix-UI tot iemand 'm vult.

Klik-flow vanuit een cel

  1. Klik cel → modaal "Welk type content?" met 6 opties (zie hieronder)
  2. Kies type → editor opent met: domain + level + type prefilled, hook-template-picker open en gefilterd op gekozen type
  3. In editor: tab-bar bovenin om type te wisselen zonder context-loss (hetzelfde verhaal naar LinkedIn én blog vertalen, sluit aan bij Phase 2.5 AI-transform)
  4. Bij type-wissel: hook-templates filteren mee, eerder ingevulde opener blijft staan met waarschuwing "deze hook past mogelijk niet bij het nieuwe type"

Migratie-plan

  1. Schema-uitbreiding — voeg level en priority kolommen toe aan topics, maak derivatives, audit_log, revisions tabellen. SQLite migration script.
  2. Domein-rename — map huidige seed-domeinen naar 8-domein structuur waar mogelijk. Algemeen → Identity (vermoedelijk), Markt → Positioning (vermoedelijk). Onzekerheid hier — Joshua moet bevestigen.
  3. AI-pass voor level + priority — Claude leest elk topic + context, stelt voor: domain, level (1-5), priority (1-5). CSV output: topic_id, suggested_domain, suggested_level, suggested_priority, rationale.
  4. Joshua-review — Joshua ontvangt CSV, accepteert/wijzigt in 30 min. Import script schrijft definitieve waarden naar DB.
  5. Front-end build — 4 tabs, editor changes, hook picker, audit log + undo flow.
  6. Test met 5 consultants — beta op huidige content.dxfferent.com URL, 1 week feedback.
BOTTOM LINE

Van topic-board naar strategische content-map

De grootste verandering is niet technisch maar conceptueel: de tool wordt een visuele weergave van Joshua's content-strategie, gekoppeld aan de Dxfferent MSP Journey. Consultants zien in één blik waar gaten zijn (lege cellen met hoge priority) en welke cellen al verzadigd zijn. Hook-templates per type houden het schrijfwerk consistent. Audit log + undo maakt open editing veilig genoeg voor 5 mensen.

Open vragen voor Joshua

1. Domein-mapping bestaande topics
Mappen "Algemeen → Identity" en "Markt → Positioning" — zijn deze aannames OK, of is een andere indeling beter? Het AI-mapping script geeft een voorstel, jij keurt af/aan.
2. Priority-data nu fake — wanneer jouw input?
MVP draait met fake priority (2-3 = hoog). Wil je direct een admin-modus om per cel een score te zetten, of bouwen we 'm later? Mijn voorkeur: later, eerst valideren of de matrix-UI überhaupt werkt voor de consultants.
3. Audit log: alleen admin-zichtbaar of voor iedereen?
Iedereen kan editen, dus iedereen zou moeten kunnen zien wie wat deed (transparantie). Of voorkeur dat alleen jij/Andy de log ziet?
4. "+ Andere type" knop in editor — wel of niet?
De zes content-types staan vast. Maar wat als een consultant een nieuwe format wil (bv. e-mail nieuwsbrief, podcast script)? Lock op 6, of "+ ander" met optionele uitbreiding?
5. LinkedIn API performance-sync
MVP is handmatig. LinkedIn API approval duurt 2-3 weken. Plannen we dat nu in als follow-up, of wachten we tot consultants vragen om automatisering?
