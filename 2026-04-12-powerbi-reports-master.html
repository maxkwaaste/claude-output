1. The Goal

Get all 940 report pages to a state where they can be bulk-pushed to proxuma.io and immediately rank for their target keywords. This means two things need to be perfect:

Design / Visual Quality

  • Every page matches the GOLDEN-STANDARD template
  • All CSS consistent (prx-report namespace)
  • Morph bar, search hero, pipeline, report overview, teal frame, printed document, CTA, JSON-LD
  • Responsive on mobile
  • No broken SVGs, missing fonts, or layout glitches
  • NL pages have Dutch UI chrome (not just translated content)

SEO Quality

  • Title tags optimized with primary keyword first
  • Meta descriptions unique, 150-160 chars, with data points
  • Hreflang EN/NL pairing on every page
  • Canonical URLs correct
  • FAQPage JSON-LD with real Q&A (not CTA text)
  • BreadcrumbList JSON-LD with correct hierarchy
  • Open Graph + Twitter Card tags
  • Related reports (6 per page, category-based)
  • No internal 404s, no missing trailing slashes
  • NL content fully translated (no English fragments)

4. Golden Standard Template Spec

Reference file: GOLDEN-STANDARD.html (CSAT Score per Client report). Every page must match this structure:

HTML Head Requirements

ElementPattern

<meta description>Unique, 150-160 chars, cites specific data or outcome
<link canonical>https://proxuma.io/powerbi/insights/{slug}/
<link hreflang en>https://proxuma.io/powerbi/insights/{slug}/
<link hreflang nl>https://proxuma.io/nl-powerbi/ai-gegenereerde-rapporten/{nl-slug}/
<meta og:title>Same as title tag (without "| Proxuma Power BI" suffix)
<meta og:description>Same as meta description
<meta og:image>https://proxuma.io/wp-content/uploads/powerbi-report-og.png (TBD)
<meta og:url>Same as canonical
<meta og:type>article
<meta twitter:card>summary_large_image
<meta twitter:site>@proxuma_io (or correct handle)
Google FontsDM Serif Display, JetBrains Mono, Open Sans

Body Structure (top to bottom)

  1. Morph bar -- fixed top bar, teal #115e58, z-index 10000, shows on scroll
  2. Series nav -- breadcrumb: Power BI > AI-Powered Reports > [Category]
  3. Search hero -- blue dotted bg, search bar with typed question, H1, subtitle, source badges
  4. Pipeline section -- 4-step pipeline (Ask Question > Connect > AI Report > Insights)
  5. Report overview -- 2-col: narrative left, meta card right (source, records, date, refresh)
  6. Teal frame -- green gradient frame containing the "printed document"
  7. Printed document -- white doc with header, title, sections, KPIs, tables, DAX toggles, findings
  8. FAQ section -- accordion Q&A (doc-faq class)
  9. Related reports -- 6 cards in 3x2 grid (rr-card class), category badges
  10. CTA section -- demo booking or contact
  11. JSON-LD -- BreadcrumbList + FAQPage (only if real FAQs exist)
