The problem

Cheops Technology is a Belgian MSP with roughly 200 employees. They use a presales Calculator (Excel, 22 tabs) to estimate hours and pricing for projects. That data then gets manually retyped into Autotask, losing structure along the way.

Core issues
  • No phase structure in Autotask. Projects are flat task lists.
  • Inconsistent task names. Presales reinvents naming every time.
  • Tasks too coarse. What Cheops calls "tasks" are really work packages. The actual tasks live as notes or checklist items, not plannable AT tasks with hours.
  • Manual copy-paste. Calculator has all project data (phases, tasks, hours, roles) but it gets hand-copied into Autotask.
  • 5 missing AT fields. Start Time, Department, Priority, Status and Task Type are absent from the Calculator format.

The solution

A set of Python scripts that read the Calculator Excel, auto-detect filled projects, apply Cheops-specific Autotask defaults, calculate sequential dates (workdays only), and produce import-ready CSV files. One command per project.

Three conversion approaches

1. Direct export
calculator_to_csv.py
Reads the Project Services. tab as-is. Phases and tasks map 1:1 to Autotask structure. Best for projects where the Calculator already has detailed task breakdowns.
2. Upscale split
upscale_split.py
Splits comma-separated subtasks in column C into individual AT tasks. Hours get distributed equally (rounded to 0.5h, minimum 0.5h per task). Turns coarse work packages into granular, plannable tasks.
3. Client workpackages
upscale_client.py
Reads a standalone client presales Excel (flat structure: Fase | Taak | Uren | Rol | Uitvoering). Tasks marked "Klant" get 0 hours but are still created for visibility in Autotask.

Technical details

Feature How it works
Auto-detect Scans for Project Planning: headers and Onboarding Managed Services blocks. Finds the data rows between header and totals row automatically, regardless of Calculator version.
Date calculation Sequential workday scheduling. Each task starts the workday after the previous task ends. Phase dates wrap their tasks. Zero-hour tasks span the full phase.
Role mapping Rol-Rate 1,2 = Project Services. Rol-Rate 3 = Project Management & Coordination. Overtime hours get a separate task line with Work Type "Project Services Overtime 150%".
AT defaults Department: Projects. Task Category: Standard. Priority: Standaard. Task Type: Fixed Work. Status: Te plannen (or New for 0-hour tasks).
CSV format 25 columns, comma-separated, MM/DD/YYYY dates, UTF-8. Matches the Autotask project import specification exactly.

Presentation talking points

Problem framing: "Every project starts in the Calculator, but gets lost in translation on the way to Autotask. Manual copy-paste means no phase structure, inconsistent naming, and coarse tasks that cannot be scheduled."

Solution pitch: "One command converts a filled Calculator into an import-ready CSV. Phases, tasks, hours, roles, dates, all mapped to Autotask format. Three approaches depending on how detailed the Calculator is filled in."

Value: "No more manual retyping. Consistent task naming from templates. Proper phase structure from day one. Plannable tasks with hours and sequential dates. And it works with any Calculator version because it auto-detects the project blocks."

Scope: "Originally estimated at 10 hours. Delivered: 4 scripts, 5 templates, 3 guides, full test suite, and a VBA macro version for users who prefer staying in Excel."
