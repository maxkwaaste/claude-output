What is this?

The SDR module adds an automated sales email pipeline to Forge. Every weekday morning at 07:00, a Claude scheduled agent will read CRM data through these API endpoints, draft 25 personalized emails, and push them into a batch. Max gets a Telegram notification with a link to the review page. He approves, edits, or skips each draft. Approved emails go out via MS Graph. Every evening, a second Claude agent reads the day's feedback (which drafts were edited, which rejected) and updates a set of "writing rules" that improve future drafts.

This implementation covers the backend only: database tables, API endpoints, artisan commands, and tests. The review UI is being built separately in Claude Design.

  • Raw SQL queries: The candidate service uses raw SQL against canonical.*, forge.*, and sdr.* schemas. The column names need validation against the actual DB.

API Endpoints

All endpoints live under /api/sdr/ and require a Sanctum token with sdr.read + sdr.write abilities.

Agent Reads (morning agent calls these)

GET /api/sdr/candidates?limit=50
SdrCandidateController@index
Returns top contacts pre-ranked by deal value and recency. Excludes contacts already emailed today, unsubscribed contacts, contacts without email. Includes last 3 email subjects, days since last email/meeting.
GET /api/sdr/candidates/{contactId}/deep-context
SdrCandidateController@deepContext
Full context for one contact: profile + enrichments, last 5 deals, last 10 emails, last 5 meetings. The agent calls this 25 times per run.
GET /api/sdr/learnings
SdrLearningController@rules
Active writing rules ordered by confidence + approval rates per category over last 7 days. The agent uses these to shape draft tone and content.
GET /api/sdr/sequences
SdrSequenceController@queued
Contacts with follow-up sequences due today (day 3 or day 7 after initial email). Includes original draft context.

Agent Writes (morning + learning agents)

POST /api/sdr/batches
SdrBatchController@store
Push a batch of up to 30 drafts. Validates category, score, contact_id (UUID). Fires Telegram notification on success. Accepts: date, complete flag, array of drafts with contact_id, category, score, subject, body, reasoning, context_used.
PUT /api/sdr/writing-rules
SdrLearningController@updateRules
Full replace of writing rules. Learning agent calls this after analyzing the day's feedback. Each rule has confidence score, example count, source (edit_diff/approve_reject/explicit).

Review Actions (called from UI)

POST /api/sdr/drafts/{id}/approve
SdrReviewController@approve
Approve or approve-with-edit. If subject/body differs from original, records as "edit" with levenshtein distance. Sends email via MS Graph. Creates day-3 follow-up sequence.
POST /api/sdr/drafts/{id}/reject
SdrReviewController@reject
Reject a draft. Optional feedback text stored for learning agent analysis.
POST /api/sdr/drafts/{id}/snooze
SdrReviewController@snooze
Snooze to next batch. Contact gets +5 score bonus next day (handled by morning agent, not this endpoint).
