Critical Fixes (Audit Findings May 5)

Two independent audits scored the account at 42-48/100. These fixes are blocking -- they corrupt data and limit reach. Do them before anything else below.

F1
Fix Conversion Action Settings (Web UI)
10 min Data corruption

Problems found:

  • "Booked a demo" (7239093290) has includeInConversionsMetric = false -- invisible to bidding and Conversions column
  • Both primary actions set to MANY_PER_CLICK -- page refreshes count as extra conversions
  • YouTube follow-on views + subscriptions set as primary conversions -- wrong signal polluting bidding

Fix in Google Ads web UI: ads.google.com > Tools & Settings > Measurement > Conversions

  1. "Booked a demo": Click > Edit Settings > Count: change "Every" to "One" > Goal: ensure "Primary action used for bidding" is selected > Save
  2. "Page view recorded-demo-requested": Click > Edit Settings > Count: "Every" to "One" > Save
  3. "YouTube follow-on views": Click > Edit Settings > Goal: select "Secondary action NOT used for bidding" > Save
  4. "YouTube channel subscriptions": Same as above -- secondary, not for bidding > Save
Cannot be done via API The Google Ads MCP doesn't support conversion action mutations. These must be done manually in the web UI. Takes 2 minutes per action.

Why: HubSpot captures the email on every demo booking. After booking, the user redirects to proxuma.io/bookingconfirmed/. The email is known but never passed to Google Ads.
