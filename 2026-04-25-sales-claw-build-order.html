Decisions Locked In

GDPR / Recording: Teams notification banner is sufficient for full compliance. No additional consent mechanism needed.
Tenant Ownership: Both Proxuma and Dxfferent tenants are Max's call. No external approval needed for admin changes.
Morning Summary: Delivered via email. That's where the day starts.
Daily Draft Cap: Start with 5, ramp up as more emails pass approval. Adaptive cap already in the spec.
AI Sending: AI creates drafts only. Max sends, whether through an app or in Outlook. AI never gets actual send access.
Consultants: Data sources, not users. They've agreed to data farming. Their calls feed content generation and AI training.
Priority: Sales Claw first. Most direct revenue impact, least blocked.
CRM Strategy: Build against Twenty CRM (where the data lives). Migrate to Forge later behind an adapter layer.
Historical Backfill: Already partially done. 161+ notes, 58 companies, 12 people enriched. Check completeness before adding more.
Definition of Done: 10+ emails/day Max would have forgotten, resulting in actual sales conversations.

CRM Reality Check

Key Finding The spec assumed Forge CRM throughout, but Twenty CRM is where the data lives. 7,603 companies, 1,891 people, 369 opportunities, 161+ call intelligence notes. NemoClaw ran against Twenty until it was paused April 14. Forge CRM schema hasn't been defined yet. Building Sales Claw against Twenty first, then migrating to Forge behind a CRM adapter, is the fastest path to value.

CRM Adapter Interface

Six methods abstract the CRM. Twenty adapter first, Forge adapter later. Same contract, swap config.

MethodWhat It Does

writeDraft(dealId, draft)Store pending draft with status
updateDraftStatus(id, status)Track approve/edit/reject/snooze + edit diff
writeLearningRecord(type, data)Edit diffs, feedback, action patterns

Build Order

1
Sales Claw MVP
Weeks 1-3 / Scoring + Drafting
1
CRM Adapter (Twenty)
Build the adapter interface against Twenty CRM API at crm.proxuma.io. Pull all 369 opportunities with contacts, stages, last activity. Pull existing call intelligence notes. Test: can we read everything we need?
2
Scoring Loop
Composite scoring: staleness + promises + momentum + inbound urgency. Run against all 369 opportunities. Output ranked JSON with score, reasoning, suggested action. Max validates: do the top 10 make sense?
3
Draft Generation
Top 5 scored deals get personalized draft emails. Uses call intelligence notes from Twenty as context. Static style template until learning loop kicks in. Drafts stored as MS365 email drafts (Max sends from Outlook).
4
Deploy on M1 Pro
Daily cron at 07:00 CET on M1 Pro (100.85.64.20). Morning email summary: "5 drafts ready for review" with deal names and scores. Telegram notification when drafts are ready. Start once/day, ramp to 3x/day later.
2
Learning Loop
Weeks 4-5 / Feedback + Adaptation
5
Feedback Capture
Track which drafts Max sends, edits, or ignores. Poll MS365 sent folder to detect sends, compare to original for edit diffs. Store in local SQLite on M1 Pro. After 20+ edit pairs, generate "Max's writing rules" document.
6
Adaptive Behavior
Adjust daily cap based on approve rate. Adjust scoring weights based on which actions Max takes. Ramp to 3x/day (07:00, 12:00, 16:30). Target: approve rate > 70%, 10+ drafts/day.
3
Transcript Pipeline
Weeks 6-8 / Auto-Capture + Extraction
7
Teams Admin Policy
Enable auto-record + auto-transcribe on both Proxuma and Dxfferent tenants. 30-minute admin change. Verify notification banner for external attendees.
8
Transcript Capture (Hybrid)
Power Automate trigger + M1 Pro processing. OneDrive folder structure by date/attendee/company. M1 Pro cron watches for new .vtt files.
9
AI Extraction
Classify calls (sales/consulting/internal). Extract pain points, objections, deal stages, action items. Write to Twenty CRM as call intelligence notes. New data feeds the scoring loop directly.
4
Playbook + Forge Migration
Weeks 9-12 / Codify + Migrate
10
Playbook Generation
Analyze full transcript library (140+ existing + new from pipeline). Sales process map, objection library, qualification criteria, product positioning. Max reviews and corrects.
11
Forge CRM Migration
Define Forge schema based on what the Twenty adapter actually uses. Build Forge adapter, swap config. Migrate data. Build the review interface in Forge (keyboard shortcuts, scoring visibility).
12
Full Integration
Playbook feeds call quality scoring. Sales Claw drafts reference playbook positioning. New calls update playbook automatically. NemoClaw fully decommissioned.

Definition of Done

Sales Claw is done when it drafts 10+ emails per day that Max would have forgotten to write, and those emails result in sales conversations.

Not "10 emails generated." 10 emails Max actually sends because the system surfaced the right prospect at the right time with the right message.
