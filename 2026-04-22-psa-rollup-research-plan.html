1. Objective & deliverables

Primary question: For each Autotask customer in the input file, is it independent or part of a larger corporate structure? If part of a group, who is the immediate parent, who is the ultimate parent, what is the ownership type (PE, public, strategic, holding), and what is the evidence?

Two deliverables produced at the end:

  1. Excel workbook — psa-ownership-<date>.xlsx in ~/ClaudeCode/market-reports/output/. Sheet 1 = master data (every account, one row, all original columns + enrichment). Sheet 2 = cluster summary (parent → children, totals). Sheet 3 = PE firm leaderboard. Sheet 4 = evidence log (one row per source hit with URL + quote).
  2. HTML report — pushed via output-page skill. Cluster map, top parents by seats and ARR, PE firm concentration, country breakdown of roll-up penetration, unclear/low-confidence list for manual review.

3. Per-entry research protocol (the core loop)

Each account runs through seven ordered probes. Short-circuit on high-confidence hit after probe 3; always do probes 6+7 for independent-looking entries to avoid false negatives.

4. Data schema (exact columns per entry)

# enrichment fields added to each row
ownership_status:   independent | subsidiary | rollup-portfolio | public-group | holding-company | unclear
parent_name:        string | null          # immediate parent
ultimate_parent:    string | null          # top of chain, e.g. "Alpine Investors" > "Evergreen"
parent_type:        pe-backed | public | strategic | conglomerate | family | unknown | na
pe_firm:            string | null          # Alpine, Oval, Thompson Street, etc.
ownership_date:     YYYY-MM-DD | null      # acquisition date if known
confidence:         high | medium | low    # high = explicit on company site / filing
research_methods:   ["homepage","about","companies_house","linkedin","crunchbase","news","pitchbook"]
evidence:           [{url, quote, source_type, retrieved_at}, ...]
research_notes:     free-form string
last_researched:    ISO datetime

Confidence rubric

  • High: statement on the company's own site, registry filing, or press release they issued. Or two independent medium sources agreeing.
  • Medium: one third-party source (Crunchbase, ChannelE2E news article, LinkedIn sidebar).
  • Low: forum post, single ambiguous snippet, inference from domain pattern only.
  • Unclear: contradictory signals or no ownership data found after all 7 probes.

# 02_crawl.py — core execution loop
import asyncio, httpx, sqlite3, json, hashlib, time
from anthropic import Anthropic
from bs4 import BeautifulSoup

CLAUDE = Anthropic()        # uses ANTHROPIC_API_KEY env
MODEL = "claude-sonnet-4-6"   # Sonnet is fine for classification
MAX_PARALLEL = 6           # tune up/down based on rate limit observations

async def research_one(entry):
    probes = []
    # Probe 1-2: site scrape
    probes.append(await fetch_site(entry["domain"], paths=HOMEPAGE_PATHS))
    # Probe 3: registry by country
    probes.append(await registry_lookup(entry["name"], entry["country"]))
    # Probe 4-7: SERP queries (gated by rate limit)
    probes.append(await serp(f'site:linkedin.com/company "{entry["name"]}"'))
    probes.append(await serp(f'site:crunchbase.com "{entry["name"]}"'))
    probes.append(await serp(f'"{entry["name"]}" acquired OR parent OR subsidiary'))
    probes.append(await serp(f'site:pitchbook.com OR site:opencorporates.com "{entry["name"]}"'))
    # Classification pass via Claude
    verdict = classify_with_claude(entry, probes)
    save_result(entry, verdict, probes)
    return verdict

def classify_with_claude(entry, probes):
    context = format_probes(probes)   # concatenate top evidence snippets
    msg = CLAUDE.messages.create(
        model=MODEL,
        max_tokens=800,
        system=CLASSIFIER_SYSTEM_PROMPT,
        messages=[{"role":"user", "content": context}]
    )
    return json.loads(extract_json(msg.content[0].text))

Classifier system prompt (exact)

You are researching Autotask PSA customer accounts to determine corporate ownership.

Given the account name, country, and a bundle of evidence snippets from the company
website, registries, LinkedIn, Crunchbase, news, and PitchBook, return a JSON verdict
with these exact fields:

  {
    "ownership_status": "independent" | "subsidiary" | "rollup-portfolio" | "public-group" | "holding-company" | "unclear",
    "parent_name": string or null,
    "ultimate_parent": string or null,
    "parent_type": "pe-backed" | "public" | "strategic" | "conglomerate" | "family" | "unknown" | "na",
    "pe_firm": string or null,
    "ownership_date": "YYYY-MM-DD" or null,
    "confidence": "high" | "medium" | "low",
    "evidence_quotes": [
      {"url": "...", "quote": "...", "source_type": "homepage"|"about"|"registry"|"linkedin"|"crunchbase"|"news"|"pitchbook"}
    ],
    "notes": string
  }

RULES:
- Prefer the company's own statements over third-party claims.
- "trading as" or "t/a" alone is a brand, not an ownership chain — flag as independent unless
  the legal entity is itself owned.
- If a known MSP roll-up parent appears (Evergreen, Lyra, Westwood, New Charter, The 20 MSP,
  BCM One Group Holdings, Constellation Software, Vela Software Group, FOG Software Group,
  Volaris, Pictet Alternative Advisors TSG, Alpine Investors, Oval Partners, Thompson Street
  Capital, PE-firm XYZ), set parent_type accordingly and fill pe_firm.
- If evidence conflicts, set "unclear" and explain in notes.
- Never invent a parent. If no evidence supports a claim, set ownership_status = "independent"
  or "unclear" (unclear only when weak signals point toward ownership but nothing confirms).
- Keep every quote verbatim from the source; never paraphrase into the quote field.

Output JSON only. No preamble.

Concurrency, caching, rate limits

  • 6 parallel workers via asyncio.Semaphore. Tune down if Google/DDG start returning CAPTCHAs.
  • Per-domain mutex — never hit the same company site twice concurrently.
  • 24h page cache in cache/pages/ keyed by URL hash. Re-runs are free.
  • SERP cache permanent, keyed by query string hash.
  • Rotating User-Agent (Chrome, Firefox, Safari recent versions).
  • Respect robots.txt for site fetches; skip and flag entry if blocked.
  • Exponential backoff on 429/403: 2s, 4s, 8s, skip after 3 retries.
  • Anthropic rate limit: Sonnet tier 4 handles 6 parallel easily; raise MAX_PARALLEL to 10 once validated.

SERP source — pick one

Three realistic options for the Google-equivalent queries. Pick at plan kickoff.

  • DuckDuckGo via ddgs Python package — free, no API key, ~1 query/sec sustainable. Recommended default.
  • Google Programmable Search Engine — 100 queries/day free, $5/1K after. Sharper results but needs CSE setup.
  • Brave Search API — 1 query/sec free tier, cleaner than DDG. Good fallback.

Resumability

Every entry's outcome writes to state.db with status = pending|in_progress|done|failed. Crash recovery: on restart, orchestrator skips done, retries failed up to 3 times, picks up pending. results.jsonl is append-only so partial progress is never lost.

7. Edge cases & failure modes

Name collisions

Multiple companies share names like "Acme Technology". Domain is the disambiguator — any time the research points at a parent, cross-check that the parent claims this domain, not just this name. Mitigation in classifier prompt: "if the parent relationship is stated for a company at a different domain than {input_domain}, lower confidence to low."

Dead domains / moved sites

Some domains 404 or redirect to a parent site — that redirect is itself evidence. Capture the redirect chain and treat the destination domain as a potential parent signal. If fully dead: flag independent + confidence low + note.

NHS trusts, universities, public bodies

A handful of entries (e.g. nelft.nhs.uk) are NHS trusts — not MSPs or software companies. Flag parent_type = "na", ownership_status = "public-sector" (extend schema), skip commercial probes.

Recent acquisitions not yet on the site

An MSP acquired last month may still show "independent" on their own site. News search (probe 6) catches this. Weight press-release evidence highly when less than 12 months old.

Staleness

Ownership changes. Record last_researched on every row. Recommend re-running top 50 every 6 months.

LinkedIn auth wall

Don't attempt to scrape LinkedIn directly. Use Google site: search — the sidebar ownership data often appears in SERP snippets. If blocked entirely, skip probe 4 and flag in research_methods.

Anti-bot on corporate sites

Some use Cloudflare bot mode. Try once with realistic headers, on 403 skip and rely on probes 3-7. Do not run a headless browser — too slow for 413+ entries.

9. HTML report spec

The final HTML uses output-page skill template. Sections, in order:

  1. Stats row — total accounts, % in a roll-up, number of distinct parent groups, top PE firm by seat count.
  2. Cluster map — grouped cards. Each card: parent name, PE firm, total seats, total ARR, child accounts listed. Sorted by seat count desc.
  3. PE firm leaderboard — horizontal bar chart, top 15 PE firms by seats controlled.
  4. Country breakdown — for each country, % of accounts in a roll-up vs independent.
  5. Unclear / low-confidence list — accounts flagged for manual review, with the ambiguity explained.
  6. Method appendix — collapsed details block with this plan's summary, research date range, cache stats.
