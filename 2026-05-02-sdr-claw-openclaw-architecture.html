4. What the Industry Says (May 2026)

The AI SDR market has matured. The "fully autonomous SDR" hype from 2024-2025 has given way to a clear consensus: hybrid human-AI works, full autonomy doesn't. Key findings from current research:

Human-in-the-loop is the winning pattern
Amplemarket, SellScale, Monday.com analysis
The top-performing AI SDR platforms in 2026 all use a hybrid model. Multiple specialized agents prepare campaigns, but a human approves in one click. Amplemarket's "Duo Copilot" (three agents, human approval) scored highest in independent evaluations. This matches your existing design: AI drafts, Max sends.
Multi-agent beats monolithic
SellScale architecture
SellScale uses Worker, Supervisor, and Secretary roles. Specialized agents outperform a single agent trying to do everything. Your scoring loop / action loop / learning loop separation maps to this pattern naturally.
CRM hygiene before automation
MarketBetter, Monday.com, Salesforce consensus
Automation amplifies whatever you feed it. Bad data means bad drafts. Twenty CRM has 7,603 companies and 369 deals. The data quality there will determine draft quality. The "data hygiene first" principle is repeatedly emphasized across all 2026 evaluations.
Learning from edits is the differentiator
Original Sales Claw spec insight
Commercial AI SDR platforms don't learn from your edit patterns. They use pre-trained models with static personas. Your learning loop (draft > Max edits > diff > pattern extraction > next draft improves) is genuinely differentiated. This was always the most valuable part of the spec.

5. Proposed Architecture: Sales Claw as OpenClaw Skill

Instead of building another custom daemon, build Sales Claw as an OpenClaw skill with three cron jobs. OpenClaw handles the infrastructure. Your code handles the sales logic.

Component 1: Scoring Cron (3x daily: 07:00, 12:00, 16:30)

  • Pulls all open deals from Twenty CRM via MCP
  • Cross-references with MS365 email activity (last touch date, response patterns)
  • Scores each deal 0-100 on: staleness vs stage norms, broken promises (next steps from call notes), momentum, inbound urgency
  • Model: DeepSeek R2 for scoring logic, V4 Flash for CRM reads
  • Output: ranked deal list stored in OpenClaw memory + pushed to Canvas

Component 2: Draft Cron (follows scoring, same 3x cadence)

  • Takes top N deals from scoring (adaptive cap, starts at 5/day)
  • For each: pulls email thread, CRM notes, Open Brain profile, call transcript excerpts
  • Generates personalized email draft referencing actual conversation context
  • Model: DeepSeek V4 Pro for drafting (good writing quality at low cost)
  • Output: drafts appear in OpenClaw Canvas for review. Actions: Approve (A), Edit (E), Reject (R), Feedback (F), Snooze (S). Approved drafts sent via MS365 MCP.

Component 3: Learning Cron (every 15 min during business hours)

  • Polls MS365 Sent folder for emails matching pending drafts
  • Computes edit diff (Levenshtein + semantic) between original draft and what Max actually sent
  • Extracts patterns: tone shifts, structural changes, added/removed phrases
  • After 20+ edit pairs: generates "Max's writing rules" document
  • Model: DeepSeek V4 Pro for diff analysis, R2 for pattern synthesis
  • Output: updated style rules in OpenClaw memory, fed into draft prompt next cycle

Component 4: Transcript Pipeline (event-driven)

  • Fathom MCP integration via Composio connector (already available in ClawHub)
  • Or: Power Automate triggers on Teams recording > OneDrive folder > OpenClaw watches folder
  • New transcripts auto-classified (sales/consulting/internal) and processed
  • Model: DeepSeek R2 for extraction, V4 Flash for classification
  • Output: structured data written to Twenty CRM, Open Brain enriched

7. The ChatGPT Computer Use Question

OpenAI's computer use (Operator) and ChatGPT Agent mode are strong at browser-based tasks: navigating websites, filling forms, researching prospects on LinkedIn. OpenClaw can route specific tasks to ChatGPT when a browser is needed.

For the SDR Claw specifically, computer use is useful for:

  • Prospect research: browsing a prospect's website, reading their blog, checking their tech stack on BuiltWith
  • LinkedIn intelligence: checking a prospect's recent posts, company updates, hiring signals
  • Competitive intel: monitoring competitor websites for pricing changes, new features

It is not needed for the core loop (scoring, drafting, learning). Those tasks work entirely through APIs and MCP tools. Computer use is an enrichment layer, not a dependency. Start without it, add it when the core loop is running.

8. Claude's Role: Build and Fix, Not Run

Claude stays out of the SDR Claw runtime entirely. No Claude API calls in the cron jobs. No Claude as a fallback model. Claude's job is:

  • Build the OpenClaw skills (Claude Code with Opus 4.7 implementation agents)
  • Fix bugs when something breaks
  • Review and improve the scoring logic, draft prompts, learning patterns
  • The daily scheduled review agent (Opus auditing draft quality, architecture, methodology)

This is a clean separation. DeepSeek pays for the high-volume runtime. Claude pays for the occasional build session. You're not burning $15/M tokens on email drafts that a $0.14/M model handles just as well.

Bottom Line

OpenClaw is the daemon/messaging/cron infrastructure you tried to build from scratch with NemoClaw. DeepSeek R2 is the reasoning engine at 1/50th the cost of Claude. The Sales Claw spec from April 24 is still the right design. What changes is the runtime: from custom Node.js daemon to OpenClaw skill, from Claude/Opus to DeepSeek, from Forge CRM to Twenty CRM (where the data actually lives). The learning loop, the scoring logic, the draft review workflow, the transcript pipeline -- all of that carries forward. You're rebuilding the runtime, not the product.

9. Open Questions to Decide

  • OpenClaw on M1 Pro or M4 Air? M1 Pro is the automation server, but you type on the M4 Air. OpenClaw daemon should run on M1 Pro (always-on), but Canvas review happens wherever you are. Telegram/WhatsApp works from your phone regardless.
  • Twenty CRM vs starting fresh? Twenty has 7,603 companies but the data quality is unverified at scale. Worth running a hygiene pass before connecting the scoring loop.
  • Fathom vs raw Teams transcripts? Fathom has a Composio MCP connector ready for OpenClaw. But it's another subscription. Raw Teams transcripts via Graph API are free but require the MS365 auth that's been blocking NemoClaw.
  • ChatGPT Pro allocation: Do you already have ChatGPT Pro ($200/mo)? If yes, the computer use tasks are essentially free to add. If not, prospect research can start with DeepSeek web search until the volume justifies it.
  • HubSpot or Twenty? HubSpot is still the active CRM with real pipeline data. Twenty was set up as a replacement but the migration may not be complete. Need to decide which is source of truth for the scoring loop.
