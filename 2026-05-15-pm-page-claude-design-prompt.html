Session 1: Product UI Mockups

This prompt asks Claude Design to build 4 static mockups that accurately represent the Proxuma product interface. Every label, column, field name, and layout detail comes from the actual frontend codebase (M4rckC/proxuma-frontend, branch feature/nova-features).

Context

I need you to create 4 accurate product UI mockups for Proxuma, an MSP planning platform. These mockups will be used as hero visuals on a marketing page. Every detail below comes from the actual codebase. Do NOT add, rename, or rearrange any elements. Render exactly what is specified. Use a clean, modern SaaS aesthetic: white backgrounds, subtle gray borders (#e2e4e8), rounded corners (8px), system font stack (-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif). Text color: #2c2c2c. Brand accent: teal #0f766e for active/selected states.

Session 2: Marketing Page

Once you have the 4 mockups from Session 1, paste them as reference images into your existing Claude Design session (with brand package loaded). Then use this prompt.

Context

Design a marketing page for Proxuma's Project Management module. This replaces the current proxuma.io/project-management/ page. Match the exact visual style, layout patterns, and section rhythm of the Kanban page (proxuma.io/kanban-preview/). Same nav, same footer, same component library. The brand package is already loaded. I'm providing 4 product UI mockups as reference images. Use these EXACTLY as the hero visuals in the relevant sections. Do not redesign or reinterpret them. They are pixel-accurate representations of the real product.

Page Structure

1. Hero
  • Badge: "Core module - Used by 100+ MSPs"
  • Headline: "Every project. Every resource. One timeline."
  • Subheading: Plan, track, and deliver Autotask projects with drag-and-drop scheduling, real-time capacity views, and automated resource allocation. Built for IT service teams that run 10-50 projects simultaneously.
  • Trust indicators: Autotask | Two-way sync | 14-day free trial | EU hosted
  • CTAs: "Start free trial" (primary), "See it live" (secondary)
  • Hero visual: Use Mockup 1 (Resource Planning Screen) as the hero image, angled/floating with subtle shadow
2. Interactive mockup showcase
  • Full-width section showing Mockup 1 (Resource Planning Screen) at larger scale
  • Annotate 3-4 key areas with small callout labels: "Plan Bucket" pointing to sidebar, "Drag to schedule" pointing to a task block, "Resource capacity" pointing to a resource row, "Week / Month / Year views" pointing to the period selector
3. Module overview: "One platform, four ways to plan"
  • Same 4-module strip as the Kanban page: Resource, Projects (highlighted/active), Shifts, Kanban
  • Keep the exact same component, just highlight "Projects" instead of "Kanban"
4. Feature: "See every project phase at a glance"
  • Use Mockup 3 (WBS Breakdown Table) as the section visual
  • Caption: "Full project hierarchy with hours tracking. Drill from project to phase to individual task, all synced live from Autotask."
  • Below the mockup, 3 feature cards: A. "Nested hierarchy" - Phases contain sub-phases, tasks, and tickets. Expand and collapse with one click. 20px indent per nesting level. B. "38 columns available" - 8 shown by default (Title, Category, Type, Dates, Priority, Status, Resources). Users toggle the rest via column chooser. C. "Inline editing" - Click any editable cell to modify values directly in the table. Changes sync to Autotask in real-time.
5. Feature: "Know your team's capacity before you schedule"
  • Use Mockup 2 (Capacity View) as the section visual
  • Highlight the color coding: green (available), amber (near capacity), red (overbooked)
  • 3 feature cards below: A. "Plan Bucket" - Your unscheduled work queue. Tasks, tickets, service calls, shifts. Drag from bucket to timeline to schedule. Sort, filter, save views. B. "Capacity breakdown" - Per resource per day: Planned hours, Spare hours, Actual hours, Billable/Non-billable split. Total capacity at a glance. C. "Bulk Edit" - Select multiple items with Shift+Click. Shift dates, reassign resources, update statuses in one action. Keeps relative scheduling intact.
6. Feature: "Your projects, your templates"
  • Use Mockup 4 (Golden Template Wizard triptych) as the section visual
  • 3 template type cards: A. "Project Templates" - Reusable blueprints with phases, tasks, roles, predecessors. Accordion form: Template Details, Associated With, Status, Estimated Revenue and Cost. B. "Golden Templates" - Master templates with validation rules, formula-based hour calculations (fixed, linear, percentage), and configurable variables. 3-step wizard: select components, fill variables, preview and generate. C. "Task and Ticket Templates" - Standardized blueprints with checklists, charges, and UDFs. Stored in Proxuma, generated items sync to Autotask automatically.
7. Two-way Autotask sync
  • Same pattern as the Kanban page sync section
  • Headline: "Two-way Autotask sync. Zero manual entry."
  • Side-by-side: action in Proxuma (drag task to new date) reflected in Autotask (status + date updated)
  • Integration logos: Autotask, Halo PSA, ServiceNow, Webhooks API, SCIM/SSO
  • Sync latency indicator: "Changes sync in under 2 seconds"
8. Before/After comparison Before Proxuma:
  • Switching between Autotask tabs to check each project status
  • Manually tracking resource availability in spreadsheets
  • No visibility into capacity until engineers are overbooked
  • Creating projects from scratch every time, copying phase structures manually
  • Status meetings to figure out who is working on what
With Proxuma Project Management:
  • Every project on one interactive timeline with real-time status
  • Capacity view shows availability per resource per day, weeks ahead
  • Drag-and-drop scheduling with instant Autotask sync
  • One-click project creation from validated templates
  • Self-service visibility: stakeholders check progress without meetings
9. Stats strip
  • "73% fewer scheduling conflicts"
  • "1.8s median sync latency"
  • "2min average project setup from template"
  • "99.9% uptime"
10. Final CTA
  • Headline: "Get your project timeline running by lunchtime"
  • Subheading: "Connect Autotask, import your projects, start scheduling. Three clicks to your first drag-and-drop."
  • CTAs: "Start 14-day free trial" (primary), "Book a 20-minute demo" (secondary)

Design Notes

  • Match the Kanban page exactly: same spacing, same card components, same button styles, same section transitions
  • Product mockup images are provided as reference. Place them as-is with subtle shadow/angle treatment, do not redraw them
  • Nav: Resource, Projects (active), Shifts, Kanban, Integrations, Pricing
  • Footer: same structure as Kanban page (Modules, Integrations, Company, Resources)
  • Mobile responsive
  • Dark section backgrounds for contrast sections (sync, before/after) same as Kanban page
