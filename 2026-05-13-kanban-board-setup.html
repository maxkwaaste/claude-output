Unmapped statuses
Statuses not assigned to a column sit in the "Unmapped statuses" sidebar: Pending Completion, On Hold, Customer Responded, Autoresolved, Acknowledged, Planned, Schedule, TEST. Drag any of these into a column to include them on the board.

Column Configuration

Board Settings → Columns tab. Each column has a name, WIP limit, color, and mapped Autotask statuses. Columns are reorderable via drag.

ColumnColorWIP LimitMapped Statuses (Waiting example)
To DoLight BlueConfigurableUser-defined mapping
In ProgressAmberConfigurableUser-defined mapping
WaitingPurpleConfigurableWaiting Approval, Waiting on predecessors, Waiting Materials, Waiting Vendor, Waiting Customer
DoneTealConfigurableUser-defined mapping

10 column colors available. Users can add new columns via "+ Add column" and drag Autotask statuses from the unmapped sidebar into any column.

Swimlane Options

Board Settings → Swimlanes tab. 9 grouping options, some restricted to specific entity types.

#SwimlaneDescriptionScope
1No SwimlanesFlat column layout without groupingAll
2PriorityGroup by priority level (P1-Critical, P2-High, P3-Medium, P4-Low, Standard, Unassigned)All
3AssigneeGroup by assigned resourceAll
4ProjectGroup by projectAll
5CompanyGroup by company/customerAll
6Event typeGroup by event typeAll
7CustomerGroup by customerTicket only
8QueueGroup by queueTicket only
9DepartmentGroup by departmentTask only
Priority swimlane values
When grouped by Priority, the swimlanes are: P1 - Critical, P2 - High, P3 - Medium, P4 - Low, Standard, Unassigned. Each swimlane is collapsible.
