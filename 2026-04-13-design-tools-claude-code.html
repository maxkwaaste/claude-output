Google Stitch (Current)

1. Google Stitch 2.0

100% Free MCP Server

Google Stitch is an AI-native design tool from Google Labs, powered by Gemini 2.5 Pro. It generates full UI designs from text prompts or hand-drawn sketches. As of March 2026 it is an infinite canvas with Vibe Design, Voice Canvas, and multi-screen generation. The entire platform is free while it remains in Google Labs -- no paid plans exist yet (expected Q4 2026).

Free Tier

350 generations per month. All features included: multi-screen generation, voice-driven design, design system imports, React code export. No paid plans until Stitch exits Labs.

MCP Connection to Claude Code

The community MCP server by David East (davideast/stitch-mcp) bridges Stitch directly into Claude Code. Install with npx @_davideast/stitch-mcp init.

Three tools exposed to Claude Code:

  • build_site -- maps design screens to routes, returns HTML for each page
  • get_screen_code -- retrieves a screen's HTML content
  • get_screen_image -- downloads a screen's screenshot as base64

Key Finding Stitch is the strongest free design tool for Claude Code right now. 350 free generations per month, a working MCP server, and the ability to go from text prompt to production HTML through Claude Code in a single session. The MCP server also supports Cursor and Gemini CLI.

2. Figma MCP Server

Free Tier Limited Official from Figma

Figma's official MCP server (docs) connects Claude Code directly to Figma files. This is the only design tool with a bidirectional MCP: Claude can both read designs and write back to the canvas.

What It Can Do

CapabilityFree TierPaid
Read designs -- pull variables, components, layout data6 calls/monthUnlimited
Generate code -- from selected frames6 calls/monthUnlimited
Write to canvas -- create frames, components, auto layoutFree (beta)Usage-based (later)
Code-to-canvas -- turn live UI into editable Figma layersFree (beta)Usage-based (later)

Free Tier Verdict

The Starter plan and View/Collab seats on paid plans are limited to 6 tool calls per month. That is barely enough for a single design session. The write-to-canvas features are currently free during beta, which is the real value -- Claude can push code back into Figma as editable layers.

Workaround: claude-talk-to-figma-mcp

Community-built alternative by arinspunk (GitHub). Uses a Figma plugin + WebSocket bridge instead of the official MCP API, so it bypasses Figma's rate limits entirely. Works with any Figma account including free. Capabilities: read/analyze designs, create elements, modify styles, export assets, audit components. Supports Claude Code, Cursor, and Antigravity.

Practical Limit The official Figma MCP at 6 calls/month is unusable for regular work. But claude-talk-to-figma-mcp (community) bypasses rate limits via WebSocket -- making Figma's free tier viable after all. The beta write features on the official MCP are genuinely useful but will become paid.

3. Penpot (Open Source)

Open Source Fully Free Official MCP

Penpot is the open-source alternative to Figma. It has an official MCP server (penpot/penpot-mcp), now integrated directly into the main Penpot repository. No usage limits. MPL-2.0 license.

MCP Capabilities

  • Read design data from Penpot files -- shapes, components, tokens
  • Create and modify design elements programmatically
  • Execute arbitrary code within the Penpot Plugin environment via Plugin API
  • Connects via WebSocket through a dedicated Penpot plugin

Setup

Requires Node.js v22+. Run npm install && npm run bootstrap to launch both the MCP server and plugin server. Load the plugin in Penpot using http://localhost:4400/manifest.json, then connect Claude Code. Penpot's "design expressed as code" approach means AI agents can read design files and tokens natively.

Key Finding Penpot is the only fully free, unlimited, open-source design tool with an MCP server. No call limits, no paid tier gatekeeping. The trade-off: Penpot's ecosystem is smaller than Figma's, and the MCP server is newer (less battle-tested). Good for teams that want full control without vendor lock-in.

4. Screenshot-Driven Design Iteration

The fastest-growing design workflow in Claude Code: generate HTML/CSS, take a screenshot, have Claude analyze it visually, then iterate. This creates a closed feedback loop where Claude can self-verify its own visual output.

Browser Automation MCPs

Playwright MCP (Microsoft Official)
Free -- Open Source
Microsoft's official MCP server. 25+ tools for browser control. Uses the accessibility tree (2-5KB structured data) instead of screenshots for most interactions -- 10-100x faster. Can navigate, click, fill forms, take screenshots, generate PDFs. Works with Claude Code, Cursor, Codex CLI. The recommended choice for cross-browser automation.
Glance -- Real Browser for Claude Code
Free -- Open Source
Open-source MCP server that gives Claude Code a real Chromium browser with 30 tools. Navigate pages, take screenshots Claude can see, click buttons, fill forms, run E2E test scenarios, visual regression testing, session recording. Featured on Product Hunt. Designed specifically for the design iteration loop.
Puppeteer MCP
Free -- Open Source
Lighter-weight, Chrome/Chromium-only option. Faster startup than Playwright. Good enough if you only need Chrome and do not need cross-browser testing. Simpler setup for quick screenshot loops.
dev-browser (SawyerHood)
Free -- Claude Code Skill
A Claude Code skill that gives the agent the ability to use a web browser. Designed for design iteration: Claude writes code, opens it in the browser, checks the result, and fixes issues in a loop.

Claude Code Desktop has auto-verify built in -- when enabled, Claude automatically takes screenshots after edits, checks for errors, and confirms changes work before completing its response. This tight loop catches layout drift, overlapping elements, and visual regressions that text-only analysis misses.

5. Other Free Design Tools with MCP

Excalidraw MCP
Free -- Open Source -- GitHub
Runs a live Excalidraw canvas controllable from AI agents via MCP. Full canvas toolkit: create, edit, export diagrams. AI agents can see what they drew. Good for wireframing and quick layout sketches before moving to code. Includes a Claude Code skill alongside the MCP server.
Canva MCP
Limited -- Prompt-Only Editing
Canva has an MCP server, but it only supports editing images by prompts -- no detailed design manipulation. Not useful for the design-to-code workflow. Canva's free tier also limits export quality. Skip this one.
draw.io Official MCP
Free -- Official -- @drawio/mcp v1.2.5 (April 2026)
Official MCP server from JGraph. Claude Code can open and edit diagrams in the draw.io editor. Best for flowcharts, system diagrams, and technical architecture. Free, runs locally or in browser.
dembrandt -- Design Token Extractor
Free -- dembrandt v0.11.0 (April 2026)
MCP server that extracts design tokens and CSS information from any website. Point it at a URL, get back colors, spacing, typography, and layout tokens. Useful for reverse-engineering existing site styles into tokens Claude Code can use as a design system foundation.
v0.dev (Vercel)
Free Tier Available
Not an MCP integration, but a parallel design-to-code tool. Generate React/Tailwind components from text prompts. Can be used alongside Claude Code: design in v0, copy the component code into your project, let Claude Code integrate it. Free tier gives limited generations per day.

7. Design Prompting Techniques

Even without external design tools, Claude Code can produce strong visual output when prompted correctly. The community has converged on several patterns:

The Aesthetics System Prompt (Official Anthropic Cookbook)

Anthropic published an official cookbook entry (link) for fighting generic output. The key move: append a <frontend_aesthetics> block to your CLAUDE.md that explicitly bans defaults (Inter, Roboto, Arial, purple-on-white) and demands distinctive choices across four dimensions: typography, color/theme, motion, and backgrounds. The typography sub-prompt lists font pairings by mood (editorial, startup, technical) and demands extreme weight contrasts (100 vs 900, not 400 vs 600).

Reference-Driven Design

Upload screenshots of designs you admire. Mention spacing, mood, typography, page flow. Claude works dramatically better with visual references than with text-only descriptions. The multi-modal input is the single biggest quality lever. Shortcut: Cmd+Ctrl+Shift+4 copies a screenshot to clipboard, then Ctrl+V pastes directly into Claude Code.

Design System First

Generate design tokens (colors, spacing, typography scale, component patterns) as a SKILL.md or design system file before writing any components. This prevents the "AI slop" aesthetic. Use create-design-system (CodyPlofker) to interview yourself about your brand and auto-generate the SKILL.md, or use dembrandt MCP to extract tokens from an existing site you admire.

Anti-Patterns to Avoid

  • Do not say "make it look nice" -- specify mood, reference sites, typography choices
  • Avoid generic fonts (Inter, Roboto, Arial) -- pick distinctive typefaces
  • Do not skip the design system step -- without it, Claude defaults to safe/boring
  • Do not rely on text-only iteration -- use screenshot feedback loops

Bottom Line: Recommended Stack

For a zero-cost design workflow with Claude Code, combine three layers:

  • Design generation: Google Stitch 2.0 (350 free gens/month) with the davideast/stitch-mcp server. Use Stitch for initial design exploration, then pull the HTML into Claude Code via MCP.
  • Visual iteration: Playwright MCP or Glance for the screenshot feedback loop. Claude writes code, takes a screenshot, analyzes the result, and iterates until the design matches intent.
  • Design system guardrails: The official frontend-design skill (277K installs) plus a custom design system SKILL.md with your brand tokens. This prevents Claude from defaulting to generic output.
  • Optional: Penpot for teams that want a full Figma-like canvas with unlimited MCP access. More setup than Stitch, but fully open source and bidirectional.
