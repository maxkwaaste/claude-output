The problem with Gladstone Go

Dolphin Square positions itself as a luxury private members club in one of London's most prestigious addresses. The current app experience does not match that positioning.

What we found

  • Typos in class names ("Powerd By I C G" instead of "Powered")
  • Activity type tags mixed with day-of-week tags in search (Cardio, Monday, HIIT, Tuesday)
  • Blank loading states when searching for activities
  • Booking a squash court takes 6+ taps from the home screen
  • No visibility into how busy courts or classes are at a glance
  • Full-screen confirmation dialog for low-stakes booking actions
  • No AI, no intelligence, no personalization

Why it matters

  • Premium members expect premium digital experiences
  • Every friction point in booking is a missed class or court session
  • Gladstone is enterprise software built for councils, not luxury clubs
  • 3d Leisure uses Gladstone across 100+ sites. Your app is identical to a military leisure centre in Catterick
  • The spa already runs on a separate booking system. Members manage two apps for one venue.

What we are proposing

Three purpose-built interfaces that work as one system. Built specifically for Dolphin Square, matching your brand, your members, and your operations.

Interface 1 Member app (iOS + Android)

The member-facing mobile app. Replaces Gladstone Go entirely.

  • AI concierge — members type or speak "book squash at 10 tomorrow" and it happens. If the exact slot is available, it books immediately. If not, it shows the nearest options pre-filled. No forms, no filters, no dropdowns.
  • Smart home screen — personalized for each member. Your next booking, your favourites, class recommendations based on history. Not a generic carousel.
  • One-tap class booking — real-time timetable with live capacity indicators. Tap a class, confirm, done. Two taps, not six.
  • Court booking — visual grid showing all 3 squash courts + tennis at a glance. Drag to select time. See what is taken, what is free.
  • QR + barcode entry — rotating QR code with 5-minute expiry for secure entry. Backwards compatible with existing barcode card scanners.
  • Reformer Pilates — separate payment flow for premium classes, handled seamlessly in the same app.
  • Push notifications — class reminders, cancellation alerts, waitlist updates.
Interface 2 Reception dashboard (web)

A web-based admin panel for front desk staff. Replaces the Gladstone back-office for day-to-day operations.

  • Member lookup — search by name, membership number, or scan their QR. Full account view: bookings, payment history, membership status.
  • Check-in log — real-time view of who is in the building. Scan verification. Guest registration.
  • Booking management — create, cancel, or modify bookings on behalf of members. Phone bookings handled instantly.
  • Class and court overview — live capacity dashboard. See at a glance which classes are full, which courts are free.
  • Account management — membership freeze, payment issues, status changes. No more logging into Gladstone's back-end.
Interface 3 AI concierge (embedded in member app)

An intelligent booking assistant powered by DeepSeek. This is not a chatbot with canned responses. It understands natural language and takes real actions.

  • "Book squash at 10 tomorrow" — checks Court 1, 2, and 3 for the 10:00 slot on 20 May. If one is free: books it, confirms. If none are free: shows the closest available times.
  • "What yoga classes are on this week?" — returns a filtered timetable: Power Yoga Wed 18:00, Vinyasa Flow Thu 07:30, Yin Yoga Sat 10:00.
  • "Cancel my squash today" — finds the booking, confirms cancellation.
  • "Am I booked for anything tomorrow?" — returns your upcoming bookings.
How it works The AI agent uses function calling against the same booking API the app uses. It can search availability, create bookings, cancel bookings, and query member schedules. It never touches payments or access control directly. When the request is ambiguous, it falls back to showing the UI rather than guessing.

Technical approach

Component Technology Why
Mobile app Expo (React Native), TypeScript One codebase, iOS + Android. Over-the-air updates without App Store review. Built-in QR scanner.
Backend Supabase (PostgreSQL) Relational database handles booking conflicts correctly. Real-time websockets for live timetable updates. Row-level security.
Payments Stripe Billing + Payment Intents Recurring memberships and one-off class purchases. Apple Pay, Google Pay. PCI compliant out of the box.
AI concierge DeepSeek API with function calling Cost-effective, fast, excellent at structured tool use. Parses natural language into booking actions.
Entry system Short-lived JWT in QR code 5-minute rotating token. Door scanner verifies signature + active membership. Works offline. Backwards compatible with barcode.
Reception dashboard Next.js web app Runs on any browser. No installation. Same Supabase backend as the mobile app.
Double-booking prevention The most common failure in booking apps is showing a slot as available then failing when two members try to book it simultaneously. We use PostgreSQL's SELECT ... FOR UPDATE SKIP LOCKED for atomic slot claiming. The database guarantees that a slot can only be claimed once, even under concurrent load. This is architecturally impossible with Firebase/Firestore, which is why we chose Supabase.

Phased delivery

We deliver in vertical slices. Each phase ships working, testable functionality. No "all the database first, then all the API, then the UI" approach.

Phase 1 — 6 weeks Core booking + entry
  • Member app: home screen, class timetable, class booking, QR entry
  • Reception dashboard: member lookup, check-in log, class overview
  • Backend: auth, booking engine, real-time timetable
  • Testable outcome: members can browse classes, book a class, and enter the building with the new app
Phase 2 — 4 weeks Courts + payments

Phase 3 — 3 weeks AI concierge + polish
