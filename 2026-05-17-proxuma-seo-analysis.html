1. Executive Summary

Five findings that matter

1. Non-branded organic is broken. 141 clicks from 152K impressions (0.09% CTR). 85% of all clicks come from people searching "proxuma." The site has visibility but fails to convert searchers who do not already know the brand.

2. Severe cannibalization. "Autotask project management" competes across 14 Proxuma pages. Google cannot pick a winner, so none ranks well. Same pattern for "autotask capacity planning" (16 pages), "autotask gantt" (10 pages), and "autotask calendar" (9 pages).

3. Report pages have visibility but zero clicks. The 449 EN report pages at /powerbi/insights/ are indexing and getting impressions, but CTR is near zero. The title tags and meta descriptions do not signal enough value for a click.

4. US market is the biggest gap. 40,699 impressions in the last 3 months, only 195 clicks (0.48% CTR). Average position is 12 -- page 2 for most queries. A few position improvements would unlock the largest English-speaking MSP market.

5. How-to guides are declining. The Autotask guide pages that drove traffic in 2025 are losing clicks: dispatch calendar -41%, open tickets -65%, line of business -46%. Meanwhile, new report/integration pages are growing from zero.

10. Thirty-Day Action Plan

Week 1 Fix Cannibalization and Title Tags
  • Pick pillar pages. For "autotask project management," designate /how-to-set-up-a-project-in-autotask/ as the canonical. For "dispatch calendar," keep the dispatch calendar guide. For "autotask capacity planning," use /project-management/.
  • Internal linking restructure. Every non-pillar page that ranks for these queries should link prominently to the pillar page. Remove competing H1/title tag overlap.
  • Title tag rewrites for top 10 pages. Current titles are descriptive but not click-optimized. Each title should follow: "[Specific Topic]: [Benefit/Outcome] for MSPs | Proxuma". Example: "Autotask Dispatch Calendar: Step-by-Step Setup Guide for MSPs."
  • Meta description rewrites. Write unique, action-oriented descriptions for the top 10 pages. Include a clear value proposition and a soft CTA.
Week 2 Create Missing Content
  • "What is Autotask PSA?" Comprehensive explainer page. 2,000+ words. Cover features, pricing overview, pros/cons, and how Proxuma enhances it. This single page could capture 1,600+ monthly impressions from "what is autotask" + "what is autotask psa" queries.
  • "Autotask Kanban Board Guide." 1,124 monthly impressions at position 14.2. No dedicated page exists. Write a step-by-step guide with screenshots showing how to set up kanban views in Autotask.
  • "Autotask vs Microsoft Planner." 1,300+ combined impressions. MSPs researching whether to use MS Planner or Autotask for project management. Position Proxuma as the bridge that makes Autotask project management work like modern PM tools.
  • Add hreflang tags to separate EN/NL report pages. The Dutch QBR page ranking for English "autotask qbr" queries is a clear signal that Google cannot distinguish the language versions.
Week 3 Report Page Template Fix
  • Redesign report page title tag template. Current pattern produces generic titles. New pattern: "[Report Name]: [What It Answers] | Proxuma for Autotask". Example: "SLA Performance Report: Track Response Times Across All Clients | Proxuma for Autotask".
  • Add meta descriptions to report pages. Currently auto-generated or missing. Each report page should have a description that answers: "What question does this report answer?" + "Why should an MSP care?"
  • FAQ schema for report pages. Add 2-3 FAQ items per report page answering common questions about the metric it covers. The 290 reports missing FAQ schema should be the priority.
  • Batch test with 20 pages first. Pick the 20 highest-impression report pages (AI privacy guide, span of control, revenue per employee, etc.). Apply the template changes. Monitor CTR for 2 weeks before rolling out to all 449.
Week 4 Competitive Content and Measurement
  • Rewrite /the-best-msp-tools/ page. Currently ranks position 63 with 61K impressions. Complete rewrite with 2026 tools, comparison tables, pricing data, and a clear Proxuma positioning section. Aim for page 2 within 90 days.
  • Update /best-psa-software-for-msps-in-2023/. The "2023" in the URL/title is hurting CTR and signals stale content to Google. Create a new page at /best-psa-software-for-msps/ with a 301 redirect from the old URL.
  • "Autotask Pricing 2026" page. 1,180+ impressions for pricing queries. No Proxuma page addresses this. Write a transparent comparison: Autotask pricing tiers vs what Proxuma adds for each tier.
  • Set up weekly GSC monitoring. Track CTR changes for the pillar pages modified in weeks 1-2. Measure whether cannibalization fixes result in improved positions within 4 weeks.

Bottom Line

Proxuma has built substantial search visibility (311K impressions) but converts almost none of it into clicks. The root causes are fixable: title/snippet optimization, cannibalization consolidation, and a report page template redesign would address 80% of the CTR gap. The highest-ROI action is fixing the cannibalization on "autotask project management" queries (14 competing pages, 1,908 impressions). The second is fixing the report page title template (one change, 449 pages improved). These two actions alone could triple non-branded organic clicks within 60 days.
