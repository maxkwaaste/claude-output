Autotask API User Setup Guide

Step-by-step instructions for creating an Autotask REST API user with the minimum permissions needed for the Proxuma MCP integration. Covers security level setup, credential generation, and the three values Claude needs.

Important Write down the Secret immediately. The API Secret (password) is shown only once when generated. If you close the form without copying it, you must regenerate it, which invalidates the old one.

What You'll Create

The Autotask REST API uses three credentials sent as HTTP headers on every request:

The Three Credentials

UserName Auto-generated 15-character key (not an email)
Secret Auto-generated 25-character password (shown once)
ApiIntegrationCode 27-character tracking ID (auto-generated, permanent)

Step 1: Create a Custom Security Level

Before creating the API user, set up a security level with minimum permissions. This avoids using the overly broad built-in "API User (system)" level.

Admin → Account Settings & Users → Resources/Users (HR) → Security Levels
  1. Find "API User (system) (API-only)" in the list
  2. Click the context menu (three dots / right-click) and select Copy
  3. Rename the copy to "Proxuma MCP Integration"
  4. Edit the permissions per the table below
  5. Click Save & Close

Why these permissions The MCP needs read access across all entities for search/query operations. Write access is limited to Projects, Tasks, Phases, Tickets, Notes, ServiceCalls, and TimeEntries. The sandbox layer in the MCP code adds a second safety gate that restricts writes to your demo resource only.

Step 2: Create the API User

Admin → Resources (Users) → click dropdown arrow next to + New → New API User

Fill in the form:

General Section

FieldValue
First NameProxuma
Last NameMCP
Email AddressYour email (for error notifications only)
ActiveChecked
LockedUnchecked
Security LevelProxuma MCP Integration (the one you just created)
Primary Internal LocationYour office location (sets timezone)

Credentials Section

  1. Click Generate Key to create the Username (15 characters)
  2. Click Generate Secret to create the Password (25 characters)
  3. Copy both values immediately and paste them somewhere safe
Critical The Secret is shown only once. After you save and close this form, you cannot view it again. If you lose it, you must regenerate (which breaks any existing integration using the old secret).

API Tracking Identifier Section

  1. Select "Custom (Internal Integration)"
  2. Enter Internal Integration Name: Proxuma MCP
  3. A tracking identifier will be auto-generated (this becomes your ApiIntegrationCode)
Permanent The tracking identifier cannot be changed after saving. Double-check the integration name before you save.

Line of Business Section

Move all items to the "Associated" pane, or check "Resource can view items with no assigned Line of Business". This ensures the integration can see all data.

Save

Click Save & Close. The API user is now active.

Step 3: Collect Your Three Values

After saving, you should have these three values. Paste them to Claude:

What to Paste to Claude

AUTOTASK_USERNAME The 15-character Key from step 2
AUTOTASK_SECRET The 25-character Secret from step 2
AUTOTASK_INTEGRATION_CODE The tracking identifier from step 2
AUTOTASK_SANDBOX_RESOURCE_NAME Your full name as it appears in Autotask (e.g. "Max de Kwaasteniet") 
AUTOTASK_USERNAME=<paste key here>
AUTOTASK_SECRET=<paste secret here>
AUTOTASK_INTEGRATION_CODE=<paste tracking ID here>
AUTOTASK_SANDBOX_RESOURCE_NAME=<your full name>

Checklist

  • Created custom security level "Proxuma MCP Integration" with minimum permissions
  • Created API User with name "Proxuma MCP"
  • Generated and saved the Username (Key)
  • Generated and saved the Secret (shown only once)
  • Selected "Custom (Internal Integration)" and named it "Proxuma MCP"
  • Associated all Lines of Business
  • Copied all four values to paste to Claude

Gotchas

IssueDetail
Secret shown onceCannot be viewed after saving. Must regenerate if lost (breaks existing integrations).
Tracking ID permanentCannot be changed after save. Each integration needs its own API user.
Rate limit sharedAll integrations on the tenant share 10,000 requests per 60-minute window. Throttling starts at 50% usage.
Silent permission blocksIf the security level lacks permission for an entity, the API returns empty results, not an error.
API users cannot log in to UIThey are API-only. They cannot be assigned to tickets/tasks as resources.
TLS 1.2 requiredAll API connections must use TLS 1.2 or higher.
