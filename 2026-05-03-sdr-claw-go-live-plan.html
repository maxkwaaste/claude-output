Estimated: 2-4 hours (fix any failures)

NeXum has strict quality gates. Every new file must pass clean. The backend CLAUDE.md defines these rules.

  • Run Pint on SDR files
    ./vendor/bin/pint --test app/Models/Sdr app/Http/Controllers/Api/Sdr app/Services/Sdr app/Console/Commands/Sdr routes/sdr.php
    Fix any violations with ./vendor/bin/pint (no --test). Claude
  • Run PHPStan Level 3 on SDR files
    ./vendor/bin/phpstan analyse app/Models/Sdr app/Http/Controllers/Api/Sdr app/Services/Sdr app/Console/Commands/Sdr --level=3
    Must be zero errors (new files don't get baseline). Claude
  • Check file sizes
    wc -l app/Services/Sdr/SdrCandidateService.php app/Services/Sdr/SdrFeedbackService.php
    SdrFeedbackService is ~120 lines, SdrCandidateService ~100. Both under 300 limit. Check longest methods individually. Claude
  • Run SDR tests
    php artisan test tests/Feature/Sdr/
    Requires a running DB with the sdr schema. Tests use real DB (no mocks for DB, as per NeXum rules). GraphMailSendService is mocked (external dependency). Claude
  • Run full test suite
    php artisan test
    Ensure SDR code doesn't break existing tests. Claude
  • Architecture review
    Verify: controllers have no business logic (delegated to services), models are clean Eloquent, services handle domain logic. SDR follows the same patterns as existing Forge modules (Billing, Marketing, etc.). Claude
  • Token scope check
    SDR routes use ability:sdr.read,sdr.write middleware. Verify this follows the TokenScopeIntersect pattern described in the backend CLAUDE.md. Never use ['*'] scope. Claude

Test End-to-End Locally
Estimated: 2-3 hours

With the app running locally on M1 Pro, test the full flow: API endpoints, review UI, email sending, feedback loop.

Outlook / MS Graph Integration

How email sending works GraphMailSendService uses Azure AD app-only auth (client_credentials) via encrypted credentials stored in canonical.platform_credentials. It sends from max@proxuma.com via the Graph API /users/max@proxuma.com/sendMail endpoint. Emails appear in Max's Outlook Sent folder. This is the same service Forge email campaigns already use on Hetzner.
PrerequisiteWhatFor Local Dev
PlatformCredentialRow in canonical.platform_credentials with platform='graph', account='proxuma'. Contains encrypted tenant_id, client_id, client_secret.Must import from Hetzner DB dump. Uses APP_KEY for decryption, so .env must have the same APP_KEY as production.
EmailSenderMailboxRow in forge.email_sender_mailboxes with email='max@proxuma.com', is_active=true.Must exist in local DB. Either imported via dump or manually seeded.
APP_KEYLaravel encryption key. PlatformCredential uses Crypt::decryptString which depends on this.Must match Hetzner's APP_KEY exactly. Different key = credentials won't decrypt = emails fail silently.
