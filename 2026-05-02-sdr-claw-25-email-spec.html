1. System Architecture

The SDR Claw skill runs as an OpenClaw cron job on M1 Pro (always-on, Tailscale-accessible). It queries Forge for all contact/deal/usage data, scores and selects the top 25 contacts, generates personalized drafts via DeepSeek, and delivers them to Max for review via Telegram inline buttons and a Canvas batch UI.

3. Forge API Mapping

Forge (NeXum-co/nexum-backend) is a Laravel 12 app with Sanctum auth. All data the SDR needs comes through its REST API. Rate limit: 120 req/min. Auth: Bearer token (personal access token with forge.read + forge.crm.update).

4. Email Categories

1. Post-Demo Follow-Up (highest priority)
Trigger: meeting transcript <5 days, no follow-up sent
Pull transcript summary, extract topics discussed, reference specific conversation moments. Warm tone, confirm next step from the call.
2. New Inbound Response
Trigger: inbound mail_signal <48h, no reply sent
Prompt response to form submissions or intro emails. Propose 15-minute call with 2 time slots.
3. Stale Deal Nudge
Trigger: active deal stage, last_signal_date >7 days
Add value (share relevant insight), ask specific question about their evaluation. Never "just checking in".
4. Customer Upsell
Trigger: PostHog usage signal (feature adoption / usage drop)
Helpful tone. "Noticed you're using X heavily, Y might help." Offer 10-minute walkthrough.
5. Win-Back
Trigger: closed_lost deal, 30-180 days ago
No pressure, lead with what's changed since they evaluated. "Worth a fresh look?"

5. Scoring Algorithm

8 weighted signals from Forge API, scored 0-10 each:

SignalWeightForge Source
Days since last email20%/contacts/{id}/timeline?types=mail
Deal stage urgency20%/deals/{id}
Days since last meeting15%/contacts/{id}/timeline?types=calendar
Deal value (log-scaled)15%/deals/{id}
Contact responsiveness10%Reply rate from timeline
Product usage signal10%/analytics/posthog/tenant/{slug}
Decision authority5%Contact enrichment
Enrichment completeness5%Contact enrichment fields filled
Selection Rule Post-demo follow-ups and new inbound responses are always included regardless of score. Remaining slots filled by highest-scored contacts from categories 3-5. Max 1 email per company per day.

6. Presidio Anonymization

Presidio runs locally on M1 Pro. Financial data is encrypted before reaching DeepSeek. Company names, contact names, and email subjects pass through for personalization.

Invoice amounts, ARR, Stripe dataEncryptFinancial PII
IBAN, BSN, credit cardsEncryptIdentity/payment PII
Phone numbersEncryptContact PII
Company names, contact namesKeepNeeded for personalization
Email addresses, subjectsKeepContext for drafting
Deal stages, usage metricsKeepBusiness context
Transcript text, notesScanFree-text gets NER scanning

Custom recognizers needed: Dutch BSN (11-proof checksum), financial amounts (EUR/USD patterns), IBAN (NL + international).

7. Review UX

Telegram (Primary)

Each draft arrives as a message with inline keyboard buttons: Send, Edit, Skip, Snooze. Edit: reply with modified text. Snooze: defer to tomorrow (+5 score bonus). Batch summary sent first with category breakdown and "Open in Canvas" link.

Canvas (Batch Mode)

HTML page at http://<tailscale-host>:18793/__openclaw__/canvas/review-queue.html. All 25 drafts as scrollable cards with editable text areas. Keyboard-driven: A=approve, E=edit, S=skip, Z=snooze, J/K=navigate, Shift+A=approve all remaining. Target: 25 drafts in under 10 minutes.

8. Learning Loop

Three Channels

Channel 1: Edit Diffs (passive, highest signal)
When Max edits a draft before sending, compute diff. After 20+ pairs, extract patterns: deleted phrases become "never use" rules, insertions become style preferences. Stored in SQLite learning.db.
Channel 2: Approve/Reject Patterns (passive)
Track correlations between actions and email attributes. After 50+ decisions: "win-back emails have 60% reject rate" informs quota allocation and template improvements.
Channel 3: Explicit Feedback (active)
When Max rejects, optionally add a reason via Telegram reply. Highest quality signal, lowest volume. Stored as labeled training data.

Daily 17:00 extraction run compiles patterns into "Max's Writing Rules" in MEMORY.md, injected into DeepSeek system prompt. Rules have confidence scores (5+ examples = low, 10+ = medium, 20+ = high). Max can view/correct via /learnings Telegram command or Canvas dashboard.

25 personalized sales emails per day for $4/month. Forge as sole data source (contacts, deals, usage, transcripts). DeepSeek for generation, Presidio for privacy, OpenClaw for orchestration. The learning loop is the differentiator: every edit Max makes teaches the system to draft better.
