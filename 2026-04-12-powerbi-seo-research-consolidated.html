1. Current State of the Repo

The GitHub repo Proxuma/powerbi-reports holds 872 HTML report files: 449 in en/ and 423 in nl/. Each file follows the naming pattern proxuma-io-SLUG-post.html. As of commit 329d666 (April 6, 2026), all files have been wrapped with a full SEO/marketing shell.

What Each Report File Contains

  • Morph bar (sticky nav)
  • Breadcrumb navigation (Home > Power BI > Category > Report)
  • Search hero section
  • Pipeline section
  • Report overview card
  • 6 related report links per page
  • CTA section
  • TechArticle + BreadcrumbList JSON-LD structured data
  • Meta descriptions, canonical URLs, hreflang EN/NL pairs
  • 17-category auto-classification with badge colors
  • NL pages have Dutch UI strings and NL URL paths

What's Still Missing or Broken

  • ~290 pages have no FAQPage JSON-LD or visible FAQ accordion (172 EN + 118 NL). The other 582 have FAQ schema preserved from raw files.
  • Breadcrumb category links point to ?cat=catId instead of real pillar page URLs
  • NL body content in ~80% of NL pages still has English narrative fragments
  • Internal links in body text on NL pages still point to EN URLs

3. Technical SEO Audit Findings

Based on the Feb/Mar 2026 audit of live WordPress pages.

What's Working

  • robots.txt allows all crawlers, links to sitemap_index.xml
  • Yoast generates canonical URLs correctly per page
  • JSON-LD: WebPage + Organization + BreadcrumbList on all pages
  • FAQPage schema on pages that have FAQ sections (9 Q&As each)
  • Internal links: ~1,048 new internal links via Related Reports blocks (was: 0)
  • Related Reports block: 6 links per page, language-isolated (EN links EN, NL links NL)

Critical Issues (from Feb 2026 audit)

IssueSeverityDetail
No meta descriptionsCriticalZero of 284 live pages had Yoast meta descriptions. Google auto-generates snippets from nav text and breadcrumbs, producing unusable SERP entries.
No hreflang tagsCriticalEN and NL pages not linked via hreflang. No WPML/Polylang/TranslatePress. Google may treat NL pages as duplicate content.
Pillar pages are thinMedium6 pillar pages exist (/ticketing/, /costs/, /projects/, /client-reports/, /upsell/, /security/) but are 6-7KB with minimal content. Risk of thin-content classification.
No homepage link to reportsMediumThe report section has no link from the homepage or site navigation. Limits crawl priority.
Status as of April 6 The April 6 batch wrap (commit 329d666) added meta descriptions, hreflang tags, and canonical URLs to all 872 repo files. But these changes need to be pushed to WordPress for the live pages to reflect them.

4. Bart's Feedback (SEO Consultant, March 12)

Bart's assessment: "Het is een behoorlijke puinhoop geworden die moet worden opgelost." (It's become quite a mess that needs fixing.) Jasper is CC'd. Priority items reordered based on Bart's input.

Bart's 6 Critical Items

1. NL body content links still point to EN URLs
Status: Related reports sections fixed. Body narrative still broken.
NL pages have hrefs matching /powerbi/insights/* in the narrative text. Should be /nl-powerbi/ai-gegenereerde-rapporten/*. Scope: ~120 NL pages. Need slug mapping derived from local files by matching report topic.
2. Trailing slashes on internal links (~400 unnecessary 301s)
Status: Not started
Links like href="/powerbi/insights/[slug]" (no trailing slash) cause 301 redirects. Need to add trailing slashes to all internal report links across ~282 pages, both EN and NL. Check both body text and Related Reports blocks.
3. Fix 404 internal links
Status: Not started
Some internal hrefs point to slugs that don't exist. Known 404s include /powerbi/insights/worst-clients and some NL slugs. Need crawl of all report pages to extract internal hrefs and verify each returns 200.
4. NL language settings broken
Status: mu-plugin exists but still broken per Bart
Need og:locale=nl_NL, <html lang="nl-NL">, and Yoast inLanguage=nl-NL in schema for all NL pages under parent 3495. The mu-plugin nl-locale-override.php may need extending.
5. NL translations (~80% have English fragments)
Status: Not started. Biggest job.
For each NL page, extract narrative text from doc-body, identify English sentences, translate to Dutch. Preserve all HTML structure, CSS, DAX queries, chart SVGs. Only translate human-readable paragraphs. Use $wpdb->update() to push. Scope: ~100 NL pages across multiple sessions.
6. Redo FAQ structured data
Status: Bad FAQ schema from Mar 12 still live on 292 pages
The March 12 FAQ injection extracted CTA text instead of real FAQ content. Step 1: Remove all existing FAQPage JSON-LD from report pages. Step 2: Only re-inject on pages that have actual <div class="doc-faq"> sections. Follow Google's FAQ structured data spec. ~290 pages also need FAQ Q&As generated and a visible accordion added.

Bart's Medium Priority Items

  • Expand related reports from 4 to 6 per page (done in April wrap)
  • Featured images with descriptive filenames + alt tags per report topic
  • Gallery coverage: ensure all 163 EN + 133 NL pages appear in galleries
  • 301 redirects for 6 root-level report URLs to /powerbi/insights/

9. Structured Data Requirements

Currently Implemented (on repo files)

  • TechArticle JSON-LD on all 872 pages
  • BreadcrumbList JSON-LD on all 872 pages
  • FAQPage JSON-LD on 582 pages (those with existing FAQ sections)

What Yoast Auto-Generates on WordPress

  • Canonical URL (correct)
  • WebPage + Organization + BreadcrumbList JSON-LD
  • og:title + og:url (correct)
  • og:description: falls back to nothing (empty)
  • hreflang: not generated (no multilingual plugin)

What Needs Fixing

  • Set _yoast_wpseo_metadesc for all pages (generated meta descriptions exist in repo files)
  • Set _yoast_wpseo_opengraph-description from the same meta descriptions
  • Inject hreflang tags via PHP filter or mu-plugin (repo files have them, WP doesn't)
  • Fix FAQPage schema on 292 pages (currently has CTA text instead of real FAQ content)
  • Generate FAQ Q&As for ~290 pages that have none
  • Update BreadcrumbList URLs to point to real pillar pages instead of ?cat=catId

11. Prioritized Action Plan for the Next Claude Session

The Goal

Perfect the 872 report files in the Proxuma/powerbi-reports GitHub repo so that when pushed to WordPress, every page has correct structured data, meta descriptions, hreflang, internal links, real FAQ content, proper NL translations, and pillar page connections. The repo is the source of truth.

Phase 1: Fix What's Broken (High Impact)

  1. Generate FAQ Q&As for ~290 pages without them. For each, create 3-5 real Q&As based on the report title, KPIs, and section content. Add both visible FAQ accordion and FAQPage JSON-LD.
  2. Fix the bad FAQ schema on 292 pages (those from Mar 12 that extracted CTA text). Remove bad schema, re-extract real Q&As from the .doc-faq sections.
  3. Fix breadcrumb links to point to real pillar page URLs (/powerbi/insights/ticketing/ etc.) instead of ?cat=catId.
  4. Fix NL body content links that still point to EN URLs. Replace /powerbi/insights/* with /nl-powerbi/ai-gegenereerde-rapporten/* in narrative text.
  5. Add trailing slashes to all internal report links.

Phase 2: Content Quality (Medium Impact)

  1. Translate English fragments in NL pages. Extract narrative text from doc-body, identify English sentences, translate to Dutch. Preserve all HTML/CSS/DAX/SVG.
  2. Verify and fix 404 internal links. Crawl all report files, extract internal hrefs, check each slug exists in the repo.
  3. Cross-reference with content planner. Map existing 872 report files to the 281 content planner entries. Identify coverage gaps (P1/P2 keywords with no matching report) and content opportunities.

Phase 3: Optimization (Refinements)

  1. Verify meta descriptions match the content planner keywords for the corresponding report.
  2. Enrich thin pages with better keyword-aligned content from the content planner (blog titles, keywords, competitor context).
  3. Check hreflang pairing between EN and NL files. Every EN file should reference its NL counterpart and vice versa.
