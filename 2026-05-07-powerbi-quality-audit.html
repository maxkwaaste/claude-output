1. Executive Summary

Key Findings 86% of reports meet STANDARD or RICH quality. The truthifier pass (8 commits, all 872 files) improved DAX and KPI coverage across the entire corpus. The remaining 14% (106 THIN + 11 SHELL + 8 CATEGORY) need targeted fixes. The gallery pages are critically stale: EN shows only 20 of 449 reports, NL shows zero NL-specific links. Hreflang cross-references are broken on 95% of files.

The audit measured every file across 20+ metrics: real DAX queries (containing EVALUATE/SUMMARIZE/CALCULATE/MEASURE), KPI values, data table cells, generic placeholder markers, structural elements, and SEO metadata. Reports were classified into five quality tiers based on content depth and placeholder contamination.

Tier Definitions

  • Rich 3+ real DAX, 4+ KPIs, 10+ data cells, zero generic markers
  • Standard 3+ real DAX, 2+ KPIs, some data. Acceptable but may contain markers
  • Thin Has some content but missing key elements (low DAX, few KPIs, or sparse tables)
  • Shell Template structure present but filled with placeholder text, few/no KPIs
  • Category Not a report at all -- a category landing page

Critical SEO Issue Hreflang cross-references are broken on 95% of files. EN files reference their own EN slug in the NL URL path (e.g., /nl-powerbi/.../alert-mttr-mean-time-to-resolve/) but the actual NL canonical uses a translated slug (/nl-powerbi/.../alert-mttr-rmm-oplostijd/). NL files do the same in reverse. Google cannot establish language pairs, which means duplicate content penalties and missed regional ranking.

Only 20 pairs could be matched (where both EN and NL use the same URL slug). The remaining ~420 pairs exist topically but use different slugs. This means:

  • No language switcher will work without a slug mapping table
  • Google cannot associate the translations
  • A mapping table needs to be built from file titles or content similarity

8. Root Cause Analysis

The quality variance has five distinct causes:

1. Multi-batch Sonnet generation with varying prompts
Reports were generated in multiple batches by Sonnet subagents. Earlier batches used simpler prompts producing SHELL-quality output (10 generic markers, zero KPIs). Later batches used report_builder.py with structured data injection. The 5 EN SHELL reports with 10 markers each (average-deal-size, average-task-cycle-time, billing-charge-allocation-conflicts, project-revenue-vs-labor-cost, psa-vs-quickbooks-comparison) are from the earliest batch.
2. Truthifier coverage gaps
The truthifier injected real DAX and KPIs from a validated library of 483 queries. Reports referencing tables not present in the demo dataset (BI_HiBob_Employees: 25 files, BI_Datto_Backup_Status: 12, BI_Microsoft_Licenses: 9) could not be enriched. These remain THIN because their topic areas have no matching DAX in the library.
3. Generic marker persistence
The truthifier injected real data but did not remove existing generic markers. 23 STANDARD/RICH reports still contain 5-10 markers ("We pulled the numbers", "Connect Your Own Data" etc.) alongside real DAX and KPIs. These need a cleanup pass to strip the filler text.
4. Category pages mixed with reports
8 files are category landing pages (beveiliging, financieel, ticketing, upsell, projecten, klantrapporten + 2 EN: power-bi-mcp-server, ai-privacy-compliance-guide). These are not Power BI reports and should be excluded from quality metrics or handled separately.
5. Hreflang generation using wrong slugs
When EN files were generated, their NL hreflang was set to the EN slug in the NL URL path. NL files did the same in reverse. This broke 95% of cross-language links. Root cause: the generation script did not look up the actual NL/EN canonical slug before setting hreflang.

9. Remediation Plan

Bottom Line

125 reports need work (14% of corpus). The fixes are mechanical and scriptable. Estimated total effort: 2-3 sessions for a Sonnet agent with proper prompts, plus 1 session for gallery regeneration and hreflang fixes.

Priority 1: Gallery Regeneration (Blocking)

Regenerate both gallery pages from the full file list. Script should:

  • Read all 449 EN + 423 NL report files
  • Extract title, slug, category from each file
  • Generate gallery HTML matching the existing card/grid format
  • Push to WP pages 3299 (EN) and 3495 (NL) via ACF field bf_sections_0_html_code

Effort: 1 session. Scriptable.

Priority 2: Strip Generic Markers (23 files)

The 23 STANDARD/RICH reports with 5+ markers need the placeholder text removed. These reports already have real data -- the markers are leftover filler. A regex-based cleanup script can handle this.

Effort: 30 minutes. Fully scriptable.

Priority 3: Enrich SHELL Reports (11 files)

The 11 SHELL reports need real DAX, KPIs, and data tables injected. 5 of these (average-deal-size, etc.) have the template structure but zero content. The other 6 are edge cases (MCP server page, duplicate SLA reports).

Effort: 1 session. Requires DAX library lookup + LLM enrichment for 5 files. The 2 MCP server pages and 2 duplicate SLA pages may be better handled by removal or redirect.

Priority 4: Upgrade THIN Reports (106 files)

These need incremental enrichment: add missing KPIs, expand tables, inject additional DAX queries. Most have 1-2 real DAX queries already -- they need 1-2 more plus KPI values.

Effort: 2-3 sessions. Can be batched by topic area.

Priority 5: Fix Hreflang Cross-References (832 files)

Build an EN-NL slug mapping table (by title similarity or manual curation). Then update all hreflang links to point to the correct counterpart slug. This is critical for SEO but does not affect visible content quality.

Effort: 1 session. Requires building the mapping first, then a script to rewrite hreflang attributes.

Priority 6: Remove/Redirect Category Pages (8 files)

The 6 NL category pages (beveiliging, financieel, etc.) and 2 EN edge cases should either be removed from the report corpus or given proper category landing page treatment. They pollute quality metrics and confuse the gallery.

Effort: 30 minutes. Decision needed on whether to keep or remove.
