Detailed Results
Recommended kokoro-tts-mcp (mitchthestonky/claude-tts-mcp)
Daily Driver Pick
Local / Free / Node.js + Python / 54 voices, 8 languages

Cleanest install of all options. Run npx claude-tts-mcp init, then claude mcp add tts -- npx claude-tts-mcp. That's it. Downloads ~200MB model on first run.

Audio quality: Natural-sounding with good prosody. Slight synthetic edge on longer passages but very listenable. Multiple voice options (American, British, Spanish, French, Japanese, Mandarin, etc.).

Latency: First run takes ~2:45 (model download + spaCy model). Warm start: 8.5s for a 75-word paragraph (29s of audio). That's 3.4x faster than real-time generation.

RAM: ~960MB during generation, drops after completion.

Why this one: Zero API keys, works offline, good quality, simple MCP integration. The auto-speak mode means Claude reads every response aloud without prompting. Speed control (0.5x-2.0x) is useful for audiobook-style listening.

free offline 54 voices auto-speak
Strong Alternative voice-mcp (shreyaskarnik/voice-mcp)
Local / Free / Python + MLX / Bidirectional (listen + speak)

The bidirectional option. Uses Apple's MLX framework for both Kokoro TTS and Voxtral Realtime STT. You can talk to Claude and Claude talks back. The listen() tool records from your mic with voice-activity detection and auto-stop.

Audio quality: Same Kokoro engine as #1, but the MLX variant. Identical quality (7/10). The real advantage is the full voice loop.

Latency: 5.3s for the same 75-word paragraph. Faster than #1 because MLX is optimized for Apple Silicon, while #1 uses PyTorch.

Gotcha: Requires Python 3.11-3.13 (3.14 breaks spaCy). First run downloads ~2.5GB of models (Voxtral STT + Kokoro TTS). Requires Apple Silicon (M1+).

free bidirectional mlx native 2.5gb models
Viable Options Cloud-backed tools
mcp-tts (blacktop/mcp-tts)
Go binary / Multi-provider / 28MB install

Tiny Go binary with 4 providers: macOS say (free, instant, robotic), ElevenLabs, Google Gemini, OpenAI. The say fallback gives you zero-config TTS right now. Add API keys later for better quality. Good architecture: concurrent playback, speed control, output directory for saving files.

tiny binary multi-provider api keys optional
cc-hooks (husniadil/cc-hooks)
Python / Hook-based / Sound effects + TTS

More than just TTS. Adds sound effects to tool execution, session lifecycle events, and completions. The free tier uses Google TTS (3.4s latency, Google Translate voice quality). Upgrade path to ElevenLabs for premium voices. Good for ambient coding feedback, not for reading long passages.

sound effects google tts free elevenlabs upgrade
claude-code-tts (ybouhjira/claude-code-tts)
Go / Hook-based / OpenAI TTS / Auto-speak

Best voice quality of all options (OpenAI's TTS voices are top-tier). Hook-based auto-speak on every response. Worker pool architecture prevents audio overlap. The catch: requires an OpenAI API key and costs ~$15 per million characters. If you have the key, this produces the most natural-sounding output.

$15/m chars best quality auto-speak
Not Recommended Broken or unavailable
kokoro-mcp-server (aparsoft)
Broken imports / Missing engine_flux module

Ambitious project with librosa audio enhancement, Streamlit UI, voice cloning, and multi-engine support. But the code is broken: ModuleNotFoundError: No module named 'aparsoft_tts.core.engine_flux'. The engine_flux module is referenced in __init__.py but doesn't exist in the repo. Dead on arrival.

broken
Soundboard Plugin (mcp-tool-shop-org)
GitHub 404 / NPM package missing

The GitHub repo returns 404 and the npm package @mcptoolshop/voice-soundboard-mcp doesn't exist. May have been taken down or was never publicly released. The concept (emotion-aware speech, SSML markup, multi-speaker dialogue) was interesting on paper but it's vaporware.

gone
VoiceMode MCP (getvoicemode.com)
Heavy install / 1.8GB / Kokoro-FastAPI backend

The most complete package (Whisper STT + Kokoro TTS + voice activity detection), but the install is brutal. Downloads ~1.8GB of dependencies including a full Kokoro-FastAPI server, PyTorch, spaCy, sudachi, and more. After 10+ minutes the service was still starting. Runs a persistent FastAPI server on port 8880. Overkill for TTS-only use. Consider it only if you need the full STT+TTS voice loop and are willing to keep a daemon running.

1.8gb full voice loop daemon required

Setup Commands
Recommended setup (kokoro-tts-mcp): 
npx claude-tts-mcp init
claude mcp add tts -- npx claude-tts-mcp
Then ask Claude to "read that back" or "say that aloud" and it will speak.
For bidirectional voice (voice-mcp): 
git clone https://github.com/shreyaskarnik/voice-mcp.git
cd voice-mcp && uv sync --python 3.12
# Add to your project's .mcp.json or global config:
claude mcp add voice -- uv --directory /path/to/voice-mcp run server.py
Tell Claude "listen to me" to activate mic input.
For instant fallback (mcp-tts with macOS say): 
go install github.com/blacktop/mcp-tts@latest
claude mcp add say -- mcp-tts
Uses macOS built-in say command. Add API keys later for better quality.

Recommendation

For daily use, install kokoro-tts-mcp as your primary TTS. It's the best balance of quality, simplicity, and zero cost. If you want to talk back to Claude (not just listen), add voice-mcp alongside it. If you ever get an OpenAI API key, claude-code-tts would be the quality upgrade path.
