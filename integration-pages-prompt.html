Each report page's HTML content contains a "Built from:" line or data source badge that names the integration. You can extract this with wp-cli + grep:

```bash
# Get all EN report page IDs
wp post list --post_type=page --post_parent=3299 --fields=ID,post_name --posts_per_page=300 --format=csv > /tmp/report-ids.csv

# For each report, extract the data source from the HTML content
while IFS=, read -r id slug; do
  [[ "$id" == "ID" ]] && continue
  source=$(wp post get "$id" --field=content 2>/dev/null | grep -oP '(?<=Built from: )[^<]+' | head -1)
  [[ -z "$source" ]] && source=$(wp post get "$id" --field=content 2>/dev/null | grep -oP 'data-source="[^"]*"' | head -1)
  echo "$slug|$source"
done < /tmp/report-ids.csv > /tmp/report-sources.csv
```

If that pattern doesn't match, look at a few report pages manually to find the actual pattern:
```bash
wp post get 6790 --field=content | grep -i 'built\|source\|integration\|autotask\|datto\|microsoft\|hubspot\|smileback\|it.glue\|n-able\|hibob\|unifi\|inforcer\|lighthouse' | head -20
```

## Publishing

Create each page as a DRAFT with noindex meta. Use wp-cli:

```bash
ssh -i ~/.ssh/id_ed25519 -p 18765 u1434-nm1gftpe9yl0@es33.siteground.eu
cd ~/www/proxuma.io/public_html

# Create the page (draft, noindex)
wp post create --post_type=page \
  --post_title="[Integration Name] Integration" \
  --post_name="[slug]" \
  --post_parent=3650 \
  --post_status=draft \
  --post_content="$(cat /tmp/integration-[slug].html)"

# Get the new page ID and add noindex
wp post meta add [NEW_ID] _yoast_wpseo_meta-robots-noindex 1
```

Generate each page's HTML to a temp file first (/tmp/integration-[slug].html), then create via wp-cli.
