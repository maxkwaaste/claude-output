Auto-Detect Row Ranges

Fix the hardcoded row ranges that break all Calculator scripts on real Cheops files. Scan for "Project Planning:" markers instead of assuming fixed positions.

The Problem

All scripts that read the Calculator's Project Services. tab use hardcoded row ranges that only match the empty REV9.34 template. In a real file (Exolon), the pricing section above the planning tables takes more rows, pushing everything down.

LocationEmpty Template (REV9.34)Exolon (Real File)Offset
Project 1 headerRow 15Row 43+28
Project 1 data startRow 19Row 47+28
Project 2 headerRow 58Row 74+16
Root cause The Calculator has two sections per project: a pricing summary (variable height) and a planning table (what we need). The pricing section grows when more roles/hours are filled in. Hardcoded ranges break because they assume the pricing section is always the same size.

Exolon Ground Truth

Scan of OFF_Exolon_Backup_DR_202604_rev3.xlsx, tab Project Services.:

rij  4: C="Project Planning: Backup infra"    -- pricing header (in C, not B)
rij  7-12: pricing calculation rows
rij 13: C="Totaal"

rij 43: B="Project Planning: Backup infra"    -- planning table header (in B!)
rij 45: B="Project Planning"                  -- subheader
rij 46: B="Omschrijving" C="Deeltaak"         -- column labels
rij 47: B="Voorbereiding"                     -- first phase
rij 48-71: tasks and phases
rij 72: C="Totaal schatting mandagen:"        -- end of block

rij 74: B="Project Planning: Project 2"       -- next project
rij 98: B="Project Planning: Project 3"       -- empty project
rij 121: B="Onboarding Managed Services"      -- MS block

Detectable Markers

  • Project header: Column B starts with "Project Planning:"
  • MS header: Column B equals "Onboarding Managed Services"
  • Column labels: B="Omschrijving" or B="Description" within 5 rows of header
  • Block end: C contains "Totaal schatting mandagen", or next header, or end of sheet

Solution: detect_project_blocks(ws)

One new function in at_shared.py that replaces all hardcoded PROJECT_RANGES dicts. Algorithm:

Step 1: Scan column B
Walk rows 1 to max_row. Collect rows where B starts with "Project Planning:" or equals "Onboarding Managed Services".
Step 2: Find data boundaries
Per header: find the column-label row (B="Omschrijving" or "Description") within 5 rows. data_start = label row + 1. data_end = first of: "Totaal schatting mandagen" row, next header - 2, or max_row.
Step 3: Return block list
Each block: {name, header_row, data_start, data_end, is_ms}. Name extracted from "Project Planning: <name>".

Scope

In scope: detect_project_blocks() in at_shared.py, refactor calculator_to_csv.py and upscale_split.py, tests with Exolon + regression tests with existing files.

Out of scope: Pre-fill template changes (separate ticket), browser tool / SheetJS (separate initiative), upscale_structured.py and upscale_client.py (not affected).
